## 1. BCE (Binary Cross-Entropy)

### 目标

最小化预测概率分布与真实标签分布之间的交叉熵，即让模型输出的概率尽可能接近真实标签（0 或 1）。

### 原理

对于二分类分割任务，逐像素计算：

$$\text{BCE} = -\frac{1}{N}\sum_{i=1}^{N}\left[ y_i \log(\hat{y}_i) + (1-y_i)\log(1-\hat{y}_i) \right]$$

其中：
- $y_i \in \{0, 1\}$：第 $i$ 个像素的真实标签
- $\hat{y}_i \in (0, 1)$：模型经过 Sigmoid 后的预测概率

当 $y_i = 1$ 时，损失约为 $-\log(\hat{y}_i)$，预测越接近 1 损失越接近 0。  
当 $y_i = 0$ 时，损失约为 $-\log(1-\hat{y}_i)$，预测越接近 0 损失越接近 0。

### 特性

| 特性 | 说明 |
|------|------|
| 对称性 | 对 FP（误检）和 FN（漏检）的惩罚权重完全相同 |
| 梯度平滑 | 概率接近 0.5 时梯度较大，不易梯度消失 |
| 概率解释 | 输出天然是 [0,1] 之间的校准概率 |
| 样本不平衡问题 | 当正负样本严重不平衡时（如道路分割中背景占 95%），大量负样本损失累积主导梯度，模型倾向于预测背景，忽略小目标 |

### 适用场景

- 样本相对平衡的分类/分割任务
- 作为组合损失的辅助项（如 D-Linknet 项目中的 `dice_bce_loss = Dice + BCE`）

### 代码实现（PyTorch）

```python
import torch.nn as nn
bce_loss = nn.BCELoss()
loss = bce_loss(pred, target)  # pred 和 target 均为 (B, 1, H, W)，值域 [0, 1]
```

## 2. Dice Loss

### 目标

直接最大化预测掩码与真实掩码之间的重叠程度（Dice 系数 / Sørensen-Dice Index）。

### 原理

Dice 系数定义为预测与真值的交集面积的两倍除以两者面积之和：

$$\text{Dice} = \frac{2 \cdot |P \cap G|}{|P| + |G|} = \frac{2 \sum_i y_i \hat{y}_i}{\sum_i y_i + \sum_i \hat{y}_i}$$

Dice Loss 定义为：

$$\mathcal{L}_{\text{Dice}} = 1 - \text{Dice}$$

实际实现中通常加入平滑项 `smooth = 1e-5` 避免除零：

$$\text{Dice}_{\text{smooth}} = \frac{2 \sum_i y_i \hat{y}_i + \epsilon}{\sum_i y_i + \sum_i \hat{y}_i + \epsilon}$$

### 特性

| 特性 | 说明 |
|------|------|
| 直接优化评估指标 | 与 IoU/Dice 评估指标直接相关，训练目标与评估目标一致 |
| 隐式处理类别不平衡 | 不显式统计正负样本数，通过交集/并集比值自动平衡，对小目标更敏感 |
| 对小目标友好 | Dice 系数对小目标的损失变化响应更明显，有利于道路细碎区域的检测 |
| 收敛震荡 | 初期可能震荡剧烈，因为小目标上的交集面积波动大 |
| 训练不稳定 | 直接优化 Dice 可能在后期陷入次优解，通常与 BCE 组合使用 |

### 适用场景

- 前景/背景严重不平衡的分割任务（医学影像、遥感道路分割）
- 与 BCE 组合使用效果通常优于单独使用

### 代码实现（PyTorch）

```python
def soft_dice_loss(y_true, y_pred, smooth=1e-5):
    intersection = (y_true * y_pred).sum()
    dice = (2. * intersection + smooth) / (y_true.sum() + y_pred.sum() + smooth)
    return 1 - dice.mean()
```

## 3. cDice (Conditional Dice)

### 目标

在 Dice 的基础上引入条件加权，改善特定结构（连通性、边界、中心区域）的分割质量。

### 原理

cDice 有多种变体，核心思想是在 Dice 公式中引入逐像素权重图 $w_i$：

$$\text{cDice} = \frac{2\sum_i w_i \cdot y_i \cdot \hat{y}_i}{\sum_i w_i \cdot (y_i + \hat{y}_i)}$$

常见的条件加权策略：

1. **中心加权**：对目标中心区域赋予更高权重，适合圆形/团块状目标
2. **边界加权**：对边界像素赋予更高权重，改善边缘模糊问题
3. **连通性加权**：对保持目标连通结构的预测进行奖励，适合道路网络
4. **距离图加权**：基于到真值边界的距离定义权重，距离越近权重越高

### 特性

| 特性 | 说明 |
|------|------|
| 保留 Dice 优势 | 继承 Dice 对不平衡场景的隐式处理能力 |
| 可控的结构偏好 | 通过权重图设计可以针对特定结构特征进行优化 |
| 计算复杂度略增 | 需要额外的权重图计算 |
| 超参数灵活 | 权重图的设计策略需要根据任务特点手工设计 |

### 适用场景

- 需要保持目标连通性的分割（道路网络、血管）
- 需要精细边界质量的场景
- 形状细长或具有特殊拓扑结构的分割任务

## 4. Focal Loss

### 目标

解决样本极度不平衡问题，同时将学习重点聚焦于难分类样本（易误检/难分类的像素）。

### 原理

由 Lin et al. (2017) 在 RetinaNet 中提出：

$$\text{FL}(p_t) = -\alpha_t (1 - p_t)^\gamma \log(p_t)$$

其中：
- $p_t = y\hat{y} + (1-y)(1-\hat{y})$，与 BCE 的概率项形式一致
- $\gamma \geq 0$：**聚焦参数**（通常设为 1, 2, 4），控制对难样本的关注程度
- $\alpha \in [0,1]$：平衡正负样本的权重因子

当 $\gamma = 0$ 时退化为普通交叉熵。$\gamma = 2$ 是常用默认值。

**核心机制**：调制因子 $(1 - p_t)^\gamma$ 对不同类型的样本产生不同效果：
- 简单负样本（$p_t$ 接近 1，即模型很有把握认为是负类）：因子接近 0，损失被大幅衰减
- 难分类样本（$p_t$ 接近 0.5，即模型不确定）：因子接近 0.5，损失几乎不被衰减
- 正样本（$y=1$，$p_t = \hat{y}$）：根据 $\alpha$ 调整权重

### 特性

| 特性 | 说明 |
|------|------|
| 易/难样本平衡 | 简单负样本贡献被压制，难样本被聚焦 |
| 动态调节 | 训练初期所有样本相对均衡；训练后期简单样本被压制，难样本主导学习 |
| 双参数调优 | $\gamma$ 控制难例聚焦程度，$\alpha$ 控制正负样本平衡 |
| 训练不稳定风险 | $\gamma$ 过大可能导致训练震荡 |

### 适用场景

- 目标检测（如 RetinaNet 的正负样本极端不平衡）
- 语义分割中的小目标或困难样本
- 当 BCE 退化、模型倾向于预测背景时

### 代码实现（PyTorch）

```python
def focal_loss(pred, target, alpha=0.25, gamma=2.0):
    bce = F.binary_cross_entropy_with_logits(pred, target, reduction='none')
    pt = torch.exp(-bce)  # p_t
    focal_weight = (1 - pt) ** gamma
    loss = alpha * focal_weight * bce
    return loss.mean()
```

## 5. Focal BCE

### 目标

结合 Focal Loss 的难例挖掘能力与 BCE 的概率校准特性，同时处理样本不平衡和难分类像素问题。

### 原理

将 Focal Loss 的调制因子 $(1-p_t)^\gamma$ 直接应用于 BCE：

$$\text{FocalBCE} = -\frac{1}{N}\sum_i \left[ \alpha (1-\hat{y}_i)^\gamma y_i \log(\hat{y}_i) + (1-\alpha)\hat{y}_i^\gamma (1-y_i)\log(1-\hat{y}_i) \right]$$

这里分开处理正样本（$y_i=1$）和负样本（$y_i=0$）的调制，更细粒度地控制两类样本的学习权重。

### 特性

| 特性 | 说明 |
|------|------|
| 难例聚焦 + 概率校准 | 继承 Focal 的难例挖掘能力，同时保留 BCE 的概率解释 |
| 双参数灵活调优 | $\alpha$ 控制正负样本平衡，$\gamma$ 控制难例聚焦程度 |
| 优于单独 BCE | 在不平衡分割任务中明显优于标准 BCE |
| 可与 Dice 组合 | 与 Dice Loss 组合使用，兼顾全局优化和难例挖掘 |

### 适用场景

- 样本不平衡的分割任务
- 需要可解释概率输出的场景
- 难分类像素占比高的分割任务

## 6. Tversky Loss

### 目标

通过调整 FP（假阳性）和 FN（假阴性）的惩罚权重，在 Dice 基础上解决对小目标/细长目标的不友好问题。

### 原理

Tversky Loss 是 Dice Loss 的广义化，引入两个参数控制假阳性和假阴性的相对惩罚：

$$\text{TI} = \frac{\sum_i y_i \hat{y}_i}{\sum_i y_i \hat{y}_i + \alpha \sum_i (1-y_i)\hat{y}_i + \beta \sum_i y_i(1-\hat{y}_i)}$$

即：

$$\mathcal{L}_{\text{Tversky}} = 1 - \frac{\text{TP}}{\text{TP} + \alpha \cdot \text{FP} + \beta \cdot \text{FN}}$$

其中：
- $\alpha$：**FN 权重**（漏检惩罚系数）
- $\beta$：**FP 权重**（误检惩罚系数）

参数含义：
- $\alpha = \beta = 0.5$：退化为 Dice/F1 形
- $\alpha = \beta = 1$：退化为 IoU/Jaccard
- $\alpha > \beta$：更怕漏检（Recall 优先）
- $\alpha < \beta$：更怕误检（Precision 优先）

### 特性

| 特性 | 说明 |
|------|------|
| FP/FN 灵活控制 | 在漏检和误检之间做精细权衡 |
| 对小目标友好 | 通过调高 $\alpha$（FN 权重），加大模型对漏检的惩罚，促进小目标检出 |
| 细长目标适用 | 道路等细长目标通常漏检多于误检，适合设置 $\alpha > \beta$ |
| 可调超参数 | $\alpha, \beta$ 需要根据任务特点调参 |

### 适用场景

- 前景极小、漏检危害大的任务
- 需要在精确率和召回率之间灵活权衡的场景
- 道路分割、血管检测等细长目标分割

### 代码实现（PyTorch）

```python
def tversky_loss(y_true, y_pred, alpha=0.7, beta=0.3, smooth=1e-5):
    TP = (y_true * y_pred).sum()
    FP = ((1 - y_true) * y_pred).sum()
    FN = (y_true * (1 - y_pred)).sum()
    tversky = (TP + smooth) / (TP + alpha * FP + beta * FN + smooth)
    return 1 - tversky
```

## 7. Focal Tversky Loss

### 目标

在 Tversky 的基础上增加对难样本的聚焦能力，同时处理：小目标问题 + FP/FN 不平衡 + 难分类像素三重挑战。

### 原理

由 Salehi et al. (2017) 在 Tversky-SGD 论文中提出：

$$\mathcal{L}_{\text{FTL}} = \left(1 - \frac{\sum_i y_i \hat{y}_i}{\sum_i y_i \hat{y}_i + \alpha \sum_i (1-y_i)\hat{y}_i + \beta \sum_i y_i(1-\hat{y}_i)} \right)^\gamma$$

即在 Tversky Loss 的基础上取 $\gamma$ 次幂（通常 $\gamma > 1$，如 $4/3, 1.5, 2$）。

**核心机制**：放大难样本（接近 0.5 的预测，即模型不确定的像素）的损失贡献：
- 当预测接近真值（简单样本）：Tversky 接近 0，$\gamma$ 次幂后更接近 0
- 当预测远离真值（困难样本）：Tversky 接近 1，$\gamma$ 次幂后被放大

### 特性

| 特性 | 说明 |
|------|------|
| 双层调制 | Tversky 调制 FP/FN 权重 + Focal 调制难易样本 |
| $\gamma$ 的作用 | $\gamma > 1$ 时放大难样本损失；$\gamma < 1$ 时压缩难样本权重 |
| 道路分割特别适合 | 道路的细碎分支和模糊边界是典型难样本，FTL 能有效聚焦 |
| 三重优化 | 同时处理：不平衡 + 小目标 + 难分类像素 |
| 超参数调优空间大 | $\alpha, \beta, \gamma$ 三个参数需要联合调参 |

### 适用场景

- 小目标分割 + 边界模糊 + 类别不平衡的综合场景
- 医学影像（肿瘤检测、脑肿瘤分割）
- 遥感道路分割（道路细碎、相互遮挡）

### 代码实现（PyTorch）

```python
def focal_tversky_loss(y_true, y_pred, alpha=0.7, beta=0.3, gamma=4/3, smooth=1e-5):
    TP = (y_true * y_pred).sum()
    FP = ((1 - y_true) * y_pred).sum()
    FN = (y_true * (1 - y_pred)).sum()
    tversky = (TP + smooth) / (TP + alpha * FP + beta * FN + smooth)
    focal_tversky = (1 - tversky) ** gamma
    return focal_tversky
```

### 常用参数参考

| 场景 | $\alpha$ | $\beta$ | $\gamma$ |
|------|---------|---------|----------|
| 标准设置 | 0.7 | 0.3 | 4/3 |
| 强召回优先 | 0.8 | 0.2 | 4/3 |
| 难例聚焦增强 | 0.7 | 0.3 | 2.0 |
| 边界敏感 | 0.6 | 0.4 | 1.5 |

## 8. 横向对比与选型建议

### 核心公式对照

| 损失函数 | 核心公式 | 可调参数 |
|---------|---------|---------|
| **BCE** | $-\sum [y\log\hat{y} + (1-y)\log(1-\hat{y})]$ | 无 |
| **Dice** | $1 - \frac{2\text{TP}}{\text{TP}+\text{FN}+\text{FP}}$ | `smooth` |
| **cDice** | Dice + 逐像素权重图 $w_i$ | 权重图设计策略 |
| **Focal** | $-\alpha(1-p_t)^\gamma\log(p_t)$ | $\alpha$, $\gamma$ |
| **Focal BCE** | $-\alpha(1-\hat{y})^\gamma y\log\hat{y} - (1-\alpha)\hat{y}^\gamma(1-y)\log(1-\hat{y})$ | $\alpha$, $\gamma$ |
| **Tversky** | $1 - \frac{\text{TP}}{\text{TP}+\alpha\text{FN}+\beta\text{FP}}$ | $\alpha$, $\beta$ |
| **Focal Tversky** | $(1 - \frac{\text{TP}}{\text{TP}+\alpha\text{FN}+\beta\text{FP}})^\gamma$ | $\alpha$, $\beta$, $\gamma$ |

### 能力矩阵

| 能力维度 | BCE | Dice | cDice | Focal | Focal BCE | Tversky | Focal Tversky |
|---------|-----|------|-------|-------|-----------|---------|---------------|
| 不平衡处理 | 弱 | 中 | 中 | 强 | 强 | 中 | 中 |
| 小目标处理 | 弱 | 中 | 强 | 中 | 中 | 强 | 强 |
| 难例挖掘 | 无 | 无 | 可选 | 强 | 强 | 无 | 强 |
| FP/FN 控制 | 无 | 无 | 可选 | 粗粒度 | 粗粒度 | 细粒度 | 细粒度 |
| 训练稳定性 | 高 | 中 | 中 | 中 | 中 | 中 | 中 |
| 边界质量 | 一般 | 一般 | 好 | 一般 | 一般 | 一般 | 好 |

### 针对道路分割的选型建议

D-Linknet 项目当前使用 `dice_bce_loss = Dice + BCE`，这是一个稳健的 baseline。

#### 升级路径

| 优先级 | 方案 | 适用情况 |
|-------|------|---------|
| **⭐⭐⭐ 推荐** | **Focal Tversky Loss** | 道路区域小且细碎、边界模糊、分支多——FTL 能同时解决小目标 + 难例 + 不平衡三重问题 |
| **⭐⭐ 进阶** | **Tversky Loss** | 只想改善小目标检出（不引入额外的 $\gamma$ 复杂度），设置 $\alpha=0.7, \beta=0.3$ |
| **⭐ 稳妥** | **Focal BCE + Dice** | 保留当前框架思路（Dice+BCE 组合），用 Focal BCE 替换标准 BCE |
| **实验性** | **cDice + 连通性权重** | 道路网络需要保持连通性时，连通性加权 cDice 特别有效 |

#### 关键超参数调参建议

**Tversky/Focal Tversky 中 $\alpha$（FN 权重）的经验法则**：

$$\alpha = \frac{\text{背景像素占比}}{1 - \text{背景像素占比}}$$

例如道路占比约 10% 时：$\alpha \approx 0.9 / 0.1 = 9$，但实践中不需要这么极端，通常 $\alpha \in [0.5, 0.8]$ 即可。

**$\gamma$ 的经验法则**（Focal Tversky）：
- 初期调试：$\gamma = 1$（退化为 Tversky，确认有效后再引入 Focal）
- 标准场景：$\gamma = 4/3$ 或 $1.5$
- 困难场景（强遮挡、模糊边界）：$\gamma = 2.0$

### 组合策略

大多数 SOTA 方法不单独使用上述损失函数，而是组合使用：

```python
# 推荐：Dice + Focal BCE（兼顾全局优化和难例聚焦）
total_loss = dice_loss + focal_bce_loss

# 进阶：Dice + Focal Tversky（针对道路分割）
total_loss = dice_loss + focal_tversky_loss

# 高阶：多任务权重自适应
total_loss = w1 * dice_loss + w2 * focal_tversky_loss + w3 * boundary_loss
```

其中权重 $w_1, w_2, w_3$ 可以通过验证集动态调整。
