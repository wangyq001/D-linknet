## 1. 植被标注形态分析

基于植被标注图（白色区域 = 植被）：

| 特征 | 描述 |
|------|------|
| **形状** | 斑块状/团块状（blob-like），非细线状 |
| **边界** | 不规则锯齿状，自然边缘 |
| **分布** | 空间稀疏，大多数区域为背景 |
| **面积** | 大小不一，多尺度（较大斑块 + 少量孤立小点） |

**对损失函数设计的直接影响**：

- 植被是**稀疏前景**问题，BCE 会被大量背景像素稀释，需要 Dice 或 Focal 类损失
- 植被的边界不规则，边界损失（如 Boundary Loss）可能有益
- 植被不需要"连通性/细线"相关的损失
- FP 误检会干扰后处理判断（把真实断裂误判为遮挡），应适当抑制

## 2. 模型架构

**双分支主-辅结构（Dual-Head DinkNet34）**

```
输入图像 (3 × 1024 × 1024)
        ↓
  ResNet34 Encoder（ImageNet 预训练）
        ↓
  D-Block（d=1,2,4,8）— 共享，不分开支
        ↓
  ┌─────────────────┬─────────────────┐
  ↓                              ↓
草线解码器                  植被解码器
（主任务分支）             （辅助任务分支）
各自独立权重                各自独立权重
        ↓                              ↓
  sigmoid → (1,H,W)       sigmoid → (1,H,W)
  草线概率图                植被概率图
```

**设计理由**：
- D-LinkNet（CVPR 2018）[论文1] 的共享 encoder 架构保证特征提取能力不因双头而退化
- LinkNet（IEEE VCIP 2017）[论文2] 的特征转发（Feature Forwarding）bypass 机制定义了两个独立解码器的 skip 连接标准
- 两个独立解码器允许每类有任务特定的特征精炼
- 独立输出互不干扰
- 与 D-LinkNet 原架构兼容（只需新增一个 head 类）

**可用的基础模型**（均可改造成双头版本）：

```
Unet / Dunet / DinkNet34 / DinkNet50 / DinkNet101 / LinkNet34 / DinkNet34_less_pool
```

## 4. 损失函数池

### 4.1 概述

损失函数以**插件式模块**实现，每个损失是一个独立的 `nn.Module`。所有损失均兼容 sigmoid 激活后的预测值（范围 [0,1]）。

| 损失名称 | 类型 | 英文名 | 论文来源 | 编号 |
|---------|------|--------|---------|------|
| Dice Loss | 交叠类 | Dice Loss | Milletari et al., 2016 — V-Net [论文3] | [3] |
| cDice Loss | 交叠类 | Generalised Dice Loss / Conditional Dice Loss | Sudre et al., 2017 — MICCAI DLMIA [论文4] | [4] |
| BCE Loss | 交叉熵类 | Binary Cross Entropy | 标准定义（无特定论文） | — |
| Focal Loss | 交叉熵类 | Focal Loss | Lin et al., 2017 — ICCV [论文5] | [5] |
| Focal BCE Loss | 交叉熵类 | Focal BCE Loss | 等价于 Focal Loss（\(\alpha=0.5\)，二值分割场景）[论文5] | [5] |
| Tversky Loss | 交叠类 | Tversky Loss | Salehi et al., 2017 — MICCAI MLMI [论文6] | [6] |
| Focal Tversky Loss | 交叠类 | Focal Tversky Loss | Abraham & Khan, 2019 — IEEE ISBI [论文7] | [7] |

> 注：论文编号对应附录A中的完整引用列表。

### 4.2 公式详解

#### 4.2.1 Dice Loss <cite>[论文3]</cite>

**思想**：直接优化预测和真值的重叠率（IoU/Jaccard），而非逐像素分类损失。首次在 V-Net 中提出，专门解决医学图像分割中前景-背景严重不平衡的问题，训练时无需手动调节样本权重。

**论文**：Milletari F, Navab N, Ahmadi S-A. *V-Net: Fully Convolutional Neural Networks for Volumetric Medical Image Segmentation*. arXiv:1606.04797, 2016. （引用 5,777+ 次）

**公式**：

Dice Coefficient（系数，非损失）：

\[
D = \frac{2\sum_i y_i p_i + \epsilon}{\sum_i y_i + \sum_i p_i + \epsilon}
\]

\[
\mathcal{L}_{Dice} = 1 - D
\]

其中 \(\epsilon = 1e-7\) 防除零。

**与现有代码的一致性**（参考 `loss.py`）：

```python
# 现有实现（loss.py 第12-23行）
def soft_dice_coeff(self, y_true, y_pred):
    smooth = 0.0
    i = torch.sum(y_true)
    j = torch.sum(y_pred)
    intersection = torch.sum(y_true * y_pred)
    score = (2. * intersection + smooth) / (i + j + smooth)
    return score.mean()

def soft_dice_loss(self, y_true, y_pred):
    return 1 - self.soft_dice_coeff(y_true, y_pred)
```

新实现保持完全一致的计算逻辑，只是封装为独立类。smooth 参数从 0.0 改为 1e-6 以避免极端情况下的数值不稳定。

#### 4.2.2 cDice Loss（Conditional Dice Loss / Generalised Dice Loss） <cite>[论文4]</cite>

**思想**：在类间高度不平衡的多类分割中，标准 Dice 会被大类主导。cDice（GDL）对每个类别按其 GT 体积的倒数加权，使得小类别不会被大类别主导；在极端不平衡场景（前景占比 0.02%）下仍能保持稳定训练。

**与本项目的关系**：二值分割下 cDice 等价于标准 Dice，保留此损失可为后续扩展到更多分割类别做准备。

**公式**：

对二值分割（单前景类）：

\[
\mathcal{L}_{cDice} = 1 - \frac{2 \cdot |y \cdot p|_1 + \epsilon}{|y|_1 + |p|_1 + \epsilon}
\]

与标准 Dice **在二值分割场景下完全等价**。cDice 的优势在多类分割时才体现。对于本项目的双类分割（草线 / 植被），cDice 与 Dice 等价，但保留此损失可以为后续扩展到更多类别做准备。

#### 4.2.3 BCE Loss

**思想**：逐像素独立计算交叉熵，是分割任务最基础的损失。

**公式**：

\[
\mathcal{L}_{BCE} = -\frac{1}{N}\sum_{i=1}^{N}\left[ y_i \log(p_i) + (1-y_i)\log(1-p_i) \right]
\]

**与现有代码的一致性**（参考 `loss.py`）：

```python
# 现有实现（loss.py 第10行 + 第30行）
self.bce_loss = nn.BCELoss()
bce = self.bce_loss(y_pred, y_true)  # 注意：现有代码是 y_pred, y_true 顺序
```

新实现保持完全一致的调用方式（使用 `nn.BCELoss()`）。

#### 4.2.4 Focal Loss <cite>[论文5]</cite>

**思想**：降低大量简单负样本的贡献，让模型聚焦于难例（hard examples）。Lin et al. 发现一阶段检测器精度不如两阶段检测器的核心原因是**类别极度不平衡**（前景:背景可能是 1:1000+）；通过 \((1-p_t)^\gamma\) 调制因子自动降低简单样本权重。RetinaNet 使用 Focal Loss（ResNet-101-FPN）在 COCO 上达到 test-dev AP 39.1，超越所有两阶段检测器。

**公式**：

\[
\mathcal{L}_{Focal} = -\frac{1}{N}\sum_i \left[ \alpha (1-p_i)^{\gamma} y_i \log(p_i) + (1-\alpha) p_i^{\gamma} (1-y_i) \log(1-p_i) \right]
\]

其中：

| 参数 | 作用 | 推荐值 |
|------|------|--------|
| \(\alpha \in [0,1]\) | 平衡正负样本权重，\(\alpha\) 越大正样本权重越高 | 0.25 |
| \(\gamma \geq 0\) | **聚焦参数**（核心），\(\gamma=0\) 时退化为普通 BCE | 2.0 |

**聚焦机制**：

| 像素类型 | 预测 p | 前景概率 \((1-p)^{\gamma}\)（以 \(\gamma=2\) 为例） | 实际贡献 |
|---------|--------|------------------------------------------------|---------|
| 简单负样本（y=0, p=0.9） | 0.9 | \((0.1)^2 = 0.01\) | 衰减至 1% |
| 难例（y=1, p≈0.5） | ~0.5 | \((0.5)^2 = 0.25\) | 保持 25% |
| 简单正样本（y=1, p→1） | →1 | \((0)^2 = 0\) | 忽略 |

#### 4.2.5 Focal BCE Loss <cite>[论文5]</cite>

**公式**：

\[
\mathcal{L}_{Focal\text{-}BCE} = -\frac{1}{N}\sum_i \left[ y_i (1-p_i)^{\gamma} \log(p_i) + (1-y_i) p_i^{\gamma} \log(1-p_i) \right]
\]

**注意**：在二值分割场景下，Focal BCE 与 Focal Loss（设 \(\alpha=0.5\)）**完全等价**。Focal Loss 中的 \(\alpha\) 参数用于调节正负样本的总体权重平衡，而 Focal BCE 省略了这一参数（等价于 \(\alpha=0.5\)，即正负样本等权）。

#### 4.2.6 Tversky Loss <cite>[论文6]</cite>

**论文**：Salehi S S M, Erdogmus D, Gholipour A. *Tversky Loss Function for Image Segmentation Using 3D Fully Convolutional Deep Networks*. MICCAI MLMI Workshop 2017. arXiv: 1706.05721.

**思想**：Dice 的一般化形式，通过 \(\alpha, \beta\) 分离 FP 和 FN 的惩罚权重。当 FP 和 FN 的代价不对称时，比 Dice 更合理。在多发性硬化（MS）病灶分割上验证，\(\alpha=0.3, \beta=0.7\) 时显著提升了 \(F_2\) score、Dice 和 PR-AUC；广泛应用于 LiTS 肝脏肿瘤分割挑战赛。

**公式**：

\[
TI = \frac{TP + \epsilon}{TP + \alpha \cdot FP + \beta \cdot FN + \epsilon}
\]

\[
\mathcal{L}_{Tversky} = 1 - TI
\]

| 参数 | 含义 | 取值倾向 |
|------|------|---------|
| \(\alpha\) | FP（误检）的惩罚系数 | 越大 → 越怕 FP |
| \(\beta\) | FN（漏检）的惩罚系数 | 越大 → 越怕 FN |
| \(\alpha = \beta = 0.5\) | 退化为 Dice | — |

**\(\alpha\) vs \(\beta\) 的选择指南**：

- 肿瘤分割（漏检更危险）：\(\alpha < \beta\)
- 道路提取（误检更难清理）：\(\alpha > \beta\)
- 本项目植被（FP 会把真实断裂误判为遮挡）：\(\alpha > \beta\)

#### 4.2.7 Focal Tversky Loss <cite>[论文7]</cite>

**论文**：Abraham N, Khan N M. *A Novel Focal Tversky Loss Function with Improved Attention U-Net for Lesion Segmentation*. **IEEE ISBI 2019**, pp. 683-687. arXiv: 1810.07842. GitHub: `nabsabraham/focal-tversky-unet`.

**思想**：Tversky + Focal 的组合，专门针对小目标/细目标/稀疏前景分割。将 Focal Loss 的"难例聚焦"机制与 Tversky Loss 的"FP/FN 非对称惩罚"叠加。在 BUS 2017（乳腺超声病灶，lesion 占图像面积 4.84%）上比 baseline U-Net 的 Dice 提升 25.7%；在 ISIC 2018（21.4%）上提升 3.6%。

**公式**：

\[
TI = \frac{TP + \epsilon}{TP + \alpha \cdot FP + \beta \cdot FN + \epsilon}
\]

\[
\mathcal{L}_{Focal\text{-}Tversky} = (1 - TI)^{\gamma}
\]

**两层调制的叠加**：

```
Tversky Index → 分离 FP/FN 惩罚权重
     ↓
(1 - TI)^γ → 聚焦难例，放大难例与简单例的损失差距
```

| 层 | 作用 |
|----|------|
| Tversky \((\alpha, \beta)\) | 控制 FP vs FN 的代价权重 |
| Focal \(\gamma\) | 控制对简单样本 vs 难例的关注度 |

**\(\gamma\) 的作用（以 \(\alpha=0.4, \beta=0.6\) 为例）：

| 预测质量 | TI | \((1-TI)^1\) | \((1-TI)^{0.75}\) | \((1-TI)^{0.5}\) |
|---------|-----|-------------|------------------|------------------|
| 好 (TI=0.9) | 0.9 | 0.10 | 0.12 | 0.32 |
| 中 (TI=0.6) | 0.6 | 0.40 | 0.37 | 0.63 |
| 差 (TI=0.2) | 0.2 | 0.80 | 0.68 | 0.89 |

- \(\gamma < 1\)：难例和简单例之间的损失差距被**放大**，模型被迫聚焦于难例
- \(\gamma = 1\)：退化为普通 Tversky Loss
- \(\gamma > 1\)：简单例损失被进一步压低，但可能训练不稳定

## 5. 双任务可配置损失

### 5.1 ConfigurableDualTaskLoss

两个分支（草线 / 植被）使用完全相同的损失函数池，通过配置字典决定各自用哪个。

```python
class ConfigurableDualTaskLoss(nn.Module):
    def __init__(self, loss_config):
        """
        loss_config: {
            'grass': {
                'Dice': 1.0,
                'BCE': 1.0,
                'Focal': 0.0,
                'FocalBCE': 0.0,
                'Tversky': 0.0,
                'FocalTversky': 0.0,
                'cDice': 0.0,
            },
            'veg': {
                'Dice': 1.0,
                'BCE': 0.0,
                'Focal': 0.0,
                'FocalBCE': 0.0,
                'Tversky': 0.0,
                'FocalTversky': 0.5,
                'cDice': 0.0,
            }
        }
        权重为 0 表示不使用该损失。
        """
        super().__init__()
        self.loss_config = loss_config
        self.available_losses = {
            'Dice': DiceLoss(),
            'BCE': BCELoss(),
            'Focal': FocalLoss(),
            'FocalBCE': FocalBCELoss(),
            'Tversky': TverskyLoss(),
            'FocalTversky': FocalTverskyLoss(),
            'cDice': ConditionalDiceLoss(),
        }
```

### 5.2 返回值结构

```python
loss_dict = criterion(pred_grass, mask_grass, pred_veg, mask_veg)

# loss_dict 包含：
# 'total'       → 总损失（用于反向传播）
# 'grass'       → 草线分支总损失
# 'veg'         → 植被分支总损失
# 'grass_breakdown': {各损失名称: 数值}  草线各损失分量
# 'veg_breakdown':   {各损失名称: 数值}  植被各损失分量
```

这样可以在训练日志中同时监控每个分支下各损失的值，便于分析梯度冲突。

## 6. 训练策略

### 6.1 两阶段渐进解冻

**核心思想**：分阶段引入植被任务，避免一开始就产生梯度冲突导致草线掉点。

```
阶段1（epoch 0 ~ WARMUP_EPOCHS）：
    植被分支全部冻结
    仅更新草线分支权重
    草线性能恢复到原模型水平
        ↓
阶段2（epoch WARMUP_EPOCHS ~ TOTAL_EPOCH）：
    解冻植被分支，联合训练
    λ_v(t) 从 0 → λ_v_max 线性增长
        ↓
    植被学习，同时主任务保持主导
```

**为什么这样做有效？**

- 阶段1 保证草线的 encoder 表征不受任何植被梯度干扰
- 阶段2 以小权重引入植被，植被分支的梯度不足以压垮草线已有的表征
- 这是 Multi-Task Learning 中防止负迁移（negative transfer）的经典策略，借鉴了 DB-MTL (arXiv:2308.12029, 2023) [论文10] 和 RI-PCGrad (Applied Intelligence, 2024) [论文9] 的梯度平衡思想

### 6.2 植被损失权重 Schedule（λ_v Ramp-up）

植被分支的权重采用线性 ramp-up：

\[
\lambda_v(t) = \lambda_{v,\max} \cdot \min\left(1, \frac{t - T_{warmup}}{T_{ramp}}\right)
\]

| 参数 | 含义 | 推荐值 |
|------|------|--------|
| \(T_{warmup}\) | 植被分支冻结阶段的总 epoch 数 | 10 ~ 20 |
| \(T_{ramp}\) | 植被权重从 0 增长到最大值的 epoch 数 | 10 ~ 20 |
| \(\lambda_{v,\max}\) | 植被损失的最大权重上限 | 0.3 ~ 0.5 |

**Schedule 可视化**（以 WARMUP=15, RAMP=15, λ_max=0.4 为例）：

```
λ_v(t)
0.4 |                          ┌───── λ_max = 0.4
0.3 |                      ┌──┘
0.2 |                  ┌──┘
0.1 |              ┌──┘
  0 |__________┌──┘
    0    10   15   20   25   30   epoch
           ↑       ↑
       T_warmup   T_warmup+T_ramp
```

### 6.3 早停与学习率调度

**与现有代码完全一致的策略**（参考 `train.py` 第57-68行）：

| 策略 | 配置参数 | 推荐值 |
|------|---------|--------|
| 早停 | `EARLY_STOP_THRESHOLD` | 连续 10 个 epoch val_loss 不下降则停止 |
| LR 衰减触发 | `LR_DECAY_THRESHOLD` | 连续 5 个 epoch 不下降则衰减 |
| LR 衰减因子 | `LR_DECAY_FACTOR` | 除以 3.0 |
| LR 下界 | `LR_MIN_BOUND` | 5e-7 |

### 6.4 梯度监控与保护

借鉴 Multi-Task Learning 中的梯度裁剪策略（Multi-Task Learning Using Gradient Balance and Clipping, Electronics 2022 [论文8]），在训练循环中加入梯度范数监控：

```python
# 草线主任务的梯度范数
grad_norm_grass = torch.nn.utils.clip_grad_norm_(
    model.decoder_grass.parameters(), max_norm=1.0
)

# 植被辅助任务的梯度范数（限制不超过草线的 50%）
if epoch >= WARMUP_EPOCHS:
    grad_norm_veg = torch.nn.utils.clip_grad_norm_(
        model.decoder_veg.parameters(), max_norm=grad_norm_grass * 0.5
    )
```

- `FocalTversky(α=0.4, β=0.6, γ=0.75)`：更抑制 FP（因为误检植被会错误解释草线断裂）

### A.3 多任务学习与训练策略论文

#### A.3.1 多任务负迁移与梯度平衡

| 研究 | 内容 |
|------|------|
| **核心问题** | Multi-Task Learning（MTL）中，辅助任务可能对主任务产生负迁移（negative transfer），即学习辅助任务反而损害主任务性能 |
| **原因** | 不同任务的梯度方向和幅度不一致，共享参数同时被多个任务拉动，导致冲突 |
| **已有方法** | (1) 损失加权（loss weighting） (2) 梯度操作（gradient manipulation） (3) 多目标优化（multi-objective optimization） |

**三篇关键参考文献：**

1. **RI-PCGrad (Applied Intelligence, 2024)** — Meng F et al. *RI-PCGrad: Optimizing multi-task learning with rescaling and impartial projecting conflict gradients*. 提出重新缩放 + 投影冲突梯度策略，综合考虑权重、梯度幅度和方向来缓解梯度冲突。

2. **Multi-Task Gradient Balance (Electronics, 2022)** — Guo Y, Wei C. *Multi-Task Learning Using Gradient Balance and Clipping with an Application in Joint Disparity Estimation and Semantic Segmentation*. 将语义分割与视差估计联合学习，使用梯度裁剪防止某一任务主导共享表征；与本项目的草线+植被双任务高度相似。

3. **Dual-Balancing MTL (arXiv:2308.12029, 2023)** — 提出 DB-MTL，从损失尺度（log 变换）和梯度幅度（归一化到最大范数）两个维度同时平衡任务；验证了梯度裁剪的有效性。

**与本项目的关系**：方案一的两阶段渐进解冻 + λ_v ramp-up + 梯度裁剪策略，综合借鉴了上述三篇论文的思路。

#### A.3.2 渐进解冻（Gradual Unfreezing）在分割任务中的应用

渐进解冻策略源自深度网络微调的经典实践，在以下场景中被验证有效：
- **辅助任务分割**：先训练主任务（如道路分割）收敛，再引入辅助任务（如边缘检测）进行联合训练
- **迁移学习微调**：先冻结 encoder 只训练 decoder，再逐步解冻 encoder 上层
- **多任务道路分割**：Weng et al. 等人在道路提取综述（*Remote Sensing*, 2024）中指出，双任务联合训练时，辅助任务的引入时机和权重 schedule 对主任务影响显著
