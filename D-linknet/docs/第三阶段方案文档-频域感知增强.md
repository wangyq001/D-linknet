# 第三阶段方案文档：频域感知增强 D-LinkNet

**版本**: v1.0
**日期**: 2026-06-08
**目标**: 增强现有 DinkNet34_less_pool_DualHead 模型对草方格草线的分割精度

---

## 一、问题背景与改进动机

### 1.1 当前模型的核心瓶颈

现有 DinkNet34_less_pool_DualHead 基于 ResNet34 encoder + LinkNet decoder 架构，在草方格遥感图像分割任务上已达到草线 IoU ~0.704、植被 IoU ~0.918 的性能。经过第二阶段损失函数筛选实验，当前损失函数组合已接近最优。

**草线分割的主要瓶颈**体现为：

1. **类内不一致（Intra-category inconsistency）**：沙漠退化场景下草线纹理变化大，同一类别内像素特征差异显著，导致分割结果在同一草线区域内部出现断裂或空洞。

2. **边界模糊（Boundary ambiguity）**：退化草方格边界不清晰，与沙地背景对比度低，现有模型难以精确界定草线边缘。

3. **周期性模式未被建模**：草方格图像具有天然的**网格周期性结构**——近似等间距排列的水平和垂直草线形成正交网格。这种周期性模式是草线的本质物理特征，但现有模型仅通过空间域卷积隐式学习，未能显式建模。

### 1.2 频域信息的必要性

频域分析能够直接捕捉草方格的以下特征：

- **高频成分**（对应灰度突变）→ 草线边缘和交叉点
- **低频成分**（对应灰度平缓区域）→ 沙地背景和植被区域
- **周期性方向性**（频谱中特定方向的能量集中）→ 草线的网格排列方向

传统 CNN 的卷积核在空间域局部操作，难以建模这种**全局周期性**模式。频域变换（如傅里叶变换、小波变换）天然具有全局性，能够在单次变换中捕获整个图像的频率分布。

### 1.3 改进目标

通过引入频域感知分支，使模型能够：
1. 显式建模草方格的周期性网格结构，增强草线方向性和连通性
2. 通过高频/低频分解改善边界精度，减少类内不一致
3. 保留轻量化优势，在可接受的推理开销内实现精度提升

## 三、改进方案总览

### 3.1 架构设计原则

本方案遵循以下原则：

1. **保留 ResNet34 backbone**：不替换现有的 pretrained ResNet34 encoder，复用其强大的空间域特征提取能力
2. **保留 D-LinkNet 解码器**：decoder3/decoder2/decoder1 的结构保持不变
3. **仅新增频域分支**：在 Stem 处新增独立的频率分支，提取原始图像的频域特征
4. **渐进式集成**：按优先级分阶段实施

### 3.2 实施路径

```
阶段一（必须）: 新增频率分支（Haar DWT + EFDCA）
  → 效果预估: IoU +1.39%

阶段二（必须）: BCAM 融合（替换解码器跳跃连接处的简单相加）
  → 效果预估: IoU +0.40%

阶段三（可选）: FAL 频域感知损失（零侵入，修改 loss.py）
  → 效果预估: IoU +0.5~1.0%
```

## 四、改进点一：Haar 小波频率分支

### 4.1 改进目的

草方格图像中，草线是灰度值剧烈变化的区域，对应高频成分；沙地背景是灰度平缓的区域，对应低频成分。Haar 小波变换能够将图像分解为四个频率子带，每个子带编码不同方向的频率信息，天然匹配草方格的网格结构。

### 4.2 Haar 小波的数学原理

二维 Haar 离散小波变换（DWT）对图像 f(x,y) 的分解公式（来自 DSWFNet 论文原文）：

```
LL(x,y) = 1/2 [f(x,y) + f(x,y+1) + f(x+1,y) + f(x+1,y+1)]
            → 低频近似（整体亮度信息）

LH(x,y) = 1/2 [f(x,y) + f(x,y+1) - f(x+1,y) - f(x+1,y+1)]
            → 水平高频（垂直边缘，响应水平草线）

HL(x,y) = 1/2 [f(x,y) - f(x,y+1) + f(x+1,y) - f(x+1,y+1)]
            → 垂直高频（水平边缘，响应垂直草线）

HH(x,y) = 1/2 [f(x,y) - f(x,y+1) - f(x+1,y) + f(x+1,y+1)]
            → 对角高频（交叉点，响应方格顶点）
```

**对草方格的意义**：

| 子带 | 数学含义 | 对草方的物理意义 |
|------|---------|-----------------|
| LL | 2×2 邻域均值 | 沙地背景和植被的亮度基准 |
| LH | 水平差分（在垂直方向上差分） | **垂直边缘** → 对应水平草线 |
| HL | 垂直差分（在水平方向上差分） | **水平边缘** → 对应垂直草线 |
| HH | 对角差分 | **交叉点** → 对应方格顶点 |

DSWFNet 论文原文明确指出：

> "Notably, road information is particularly prominent in the **HL sub-bands**, which emphasize **horizontal edge structures**."

这一定论直接可迁移到草方格场景——HL 子带对水平草线具有最强的响应。

### 4.3 为什么选择 Haar 小波

Haar 小波被选用的原因（来自 DSWFNet 论文原文）：

> "The R-channel HL sub-bands are compared in Fig. 2, showing that db1 (Haar) yields **sharper road boundaries and better connectivity**, more clearly delineating thin, low-contrast road structures."

> "As the order increases, wavelets become smoother... and **exacerbates boundary effects**. Accordingly, we employ the Haar wavelet as the DWT basis: its **low computational cost**, **memory efficiency**, and strong **sensitivity to local discontinuities** (road edges) make it particularly suitable for large-scale remote sensing segmentation."

Haar 小波的四个优势：
1. **计算代价最低**：仅涉及加减运算，无复杂卷积核
2. **边界锐利**：2×2 局部差分天然保留高频边缘
3. **对局部不连续性敏感**：最适合检测草线这类细线结构
4. **无边界效应**：比 Daubechies、Symlets 等高阶小波更适合细线检测

### 4.4 实现细节

#### 4.4.1 频率分支架构

```
输入图像 (3, H, W)
    │
    ├──→ [空间分支] ResNet34 encoder → e1, e2, e3
    │
    └──→ [频率分支] Haar DWT → LL/LH/HL/HH (每通道 4 子带 × 3 通道 = 12 通道)
                │
                └──→ Conv2d(12 → 64) → 与 e1 通道数对齐
                │
                └──→ Conv2d(64 → 128) → 与 e2 通道数对齐
                │
                └──→ EFDCA 增强 → f1, f2, f3
```

#### 4.4.2 PyTorch 实现代码

```python
import torch
import torch.nn as nn
import torch.nn.functional as F


class HaarWaveletTransform2D(nn.Module):
    """
    二维 Haar 离散小波变换（DSWFNet 论文 Fig.2, 公式 1-4）

    输入: (B, C, H, W) 的 RGB 图像或特征图
    输出: 四个子带 LL, LH, HL, HH
          每个子带分辨率减半，通道数为输入的 4 倍
    """
    def __init__(self):
        super().__init__()

    def forward(self, x):
        # x: (B, C, H, W)，要求 H, W 为偶数
        assert x.size(2) % 2 == 0 and x.size(3) % 2 == 0, \
            "Haar DWT requires even spatial dimensions"

        # 下采样因子 2 的 2×2 邻域分组
        # x[:, :, 0::2, 0::2]      → 左上角
        # x[:, :, 0::2, 1::2]      → 右上角
        # x[:, :, 1::2, 0::2]      → 左下角
        # x[:, :, 1::2, 1::2]      → 右下角
        x00 = x[:, :, 0::2, 0::2]      # 左上
        x01 = x[:, :, 0::2, 1::2]      # 右上
        x10 = x[:, :, 1::2, 0::2]      # 左下
        x11 = x[:, :, 1::2, 1::2]      # 右下

        # LL: 低频近似（邻域均值）
        LL = (x00 + x01 + x10 + x11) * 0.5

        # LH: 水平高频（垂直边缘）
        # 响应垂直边缘（水平草线）
        LH = (x00 + x01 - x10 - x11) * 0.5

        # HL: 垂直高频（水平边缘）
        # 响应水平边缘（垂直草线）
        HL = (x00 - x01 + x10 - x11) * 0.5

        # HH: 对角高频（交叉点）
        HH = (x00 - x01 - x10 + x11) * 0.5

        return LL, LH, HL, HH


class FrequencyBranch(nn.Module):
    """
    频率分支（参考 DSWFNet 论文 Fig.1 右侧 Frequency Branch）

    对原始输入图像做 Haar DWT 分解，
    将 4 个子带（LL/LH/HL/HH）通过独立卷积映射到与 ResNet34 各阶段
    通道数匹配的表示空间。
    """
    def __init__(self, spatial_channels=(64, 128, 256)):
        """
        Args:
            spatial_channels: ResNet34 encoder 各阶段的通道数 (e1, e2, e3)
        """
        super().__init__()
        c1, c2, c3 = spatial_channels

        # 输入: RGB 图像 (3, H, W) → DWT 分解 → 12 通道 (每个子带 3 通道)
        # 12 → e1 通道映射
        self.map_f1 = nn.Sequential(
            nn.Conv2d(12, c1, kernel_size=1, bias=False),
            nn.BatchNorm2d(c1),
            nn.ReLU(inplace=True),
        )
        # e1 → e2 通道映射（经 EFDCA 增强）
        self.efdca_f2 = EFDCA(c1, c2)
        # e2 → e3 通道映射（经 EFDCA 增强）
        self.efdca_f3 = EFDCA(c2, c3)

        self.haar = HaarWaveletTransform2D()

    def forward(self, x_input):
        """
        Args:
            x_input: 原始输入图像 (B, 3, H, W)
        Returns:
            f1, f2, f3: 与 ResNet34 encoder 各阶段通道数对齐的频率特征
        """
        # Haar DWT 分解（对每通道 RGB）
        # x_input: (B, 3, H, W) → [B, 3, H/2, W/2] × 4 子带
        LL, LH, HL, HH = self.haar(x_input)

        # 在通道维度拼接 4 个子带
        # Concatenate along channel: (B, 3, H/2, W/2) × 4 → (B, 12, H/2, W/2)
        freq_feat = torch.cat([LL, LH, HL, HH], dim=1)

        # 映射到 e1 通道数 (64)
        f1 = self.map_f1(freq_feat)          # (B, 64, H/2, W/2)

        # 逐阶段 EFDCA 增强
        f2 = self.efdca_f2(f1)               # (B, 128, H/4, W/4)
        f3 = self.efdca_f3(f2)               # (B, 256, H/8, W/8)

        return f1, f2, f3
```

#### 4.4.3 集成到 DinkNet34_less_pool_DualHead 的 forward 流程

频率分支仅处理原始输入图像（firstconv 之前），与空间分支并行提取特征：

```python
def forward(self, x):
    # ======== 空间分支（现有 ResNet34 encoder）=======
    x_spatial = self.firstconv(x)      # (B, 64, H/4, W/4)
    x_spatial = self.firstbn(x_spatial)
    x_spatial = self.firstrelu(x_spatial)
    x_spatial = self.firstmaxpool(x_spatial)  # (B, 64, H/8, W/8)
    e1_s = self.encoder1(x_spatial)      # (B, 64, H/8, W/8)
    e2_s = self.encoder2(e1_s)          # (B, 128, H/16, W/16)
    e3_s = self.encoder3(e2_s)          # (B, 256, H/32, W/32)

    # ======== 频率分支（新增 Haar DWT）=======
    f1, f2, f3 = self.freq_branch(x)     # f1:(B,64,H/2,W/2), f2:(B,128,H/4,W/4), f3:(B,256,H/8,W/8)

    # 调整频率分支分辨率使其与空间分支各阶段对齐
    # f1: (B, 64, H/2, W/2) → e1 尺寸
    f1_aligned = F.max_pool2d(f1, kernel_size=2)  # (B, 64, H/4, W/4)
    f1_aligned = F.max_pool2d(f1_aligned, kernel_size=2)  # (B, 64, H/8, W/8)

    # f2: (B, 128, H/4, W/4) → e2 尺寸
    f2_aligned = F.max_pool2d(f2, kernel_size=2)  # (B, 128, H/8, W/8)

    # f3: (B, 256, H/8, W/8) → e3 尺寸（已是相同尺寸）

    # 频率特征与空间特征逐通道相加（或拼接后 1×1 卷积融合）
    e1 = e1_s + f1_aligned  # (B, 64, H/8, W/8)
    e2 = e2_s + f2_aligned  # (B, 128, H/16, W/16)
    e3 = e3_s + f3          # (B, 256, H/32, W/32)

    # ======== Center (现有 dblock) ========
    e3 = self.dblock(e3)

    # ======== Decoder + BCAM 融合（见第五节）=======
    d3 = self.decoder3(e3)
    d3_fused = self.bcam3(d3, e2) + e2          # BCAM 融合
    d2 = self.decoder2(d3_fused)
    d2_fused = self.bcam2(d2, e1) + e1          # BCAM 融合
    d1 = self.decoder1(d2_fused)

    # ======== 双头输出（现有逻辑）=======
    # ... 草线解码器 + 植被解码器 ...
```

## 五、改进点二：EFDCA 模块

### 5.1 改进目的

EFDCA（Enhanced Frequency-Domain Channel Attention，增强型频域通道注意力）用于在频率分支内部增强频域特征的判别力。原始 Haar DWT 分解得到的频域特征通道数较少（LL/LH/HL/HH 各 3 通道），需要通过通道注意力机制选择对草线最有判别力的频率成分，抑制不重要的高频噪声。

### 5.2 数学原理

EFDCA 基于经典 SE（Squeeze-and-Excitation）通道注意力机制改进而来（来自 DSWFNet 论文原文 Fig.5，公式 14-18）：

```
给定输入特征图 X ∈ R^(B×C×H×W)：

Step 1: 全局统计聚合
  Z_avg = AvgPool(X) ∈ R^(B×C×1×1)    # 全局平均池化
  Z_max = MaxPool(X) ∈ R^(B×C×1×1)    # 全局最大池化

Step 2: 通道描述符拼接
  Z = Concat([Z_avg; Z_max]) ∈ R^(B×2C×1×1)

Step 3: 非线性变换生成通道权重
  A = σ(W3 · ReLU(W2 · ReLU(BN(W1 · Z))))
  其中: W1: 2C → C/r,  W2: C/r → C/r,  W3: C/r → C,  r 为通道缩减比（通常 r=16）
       BN 为批归一化

Step 4: DropPath 正则化（训练时随机丢弃 10% 注意力权重，防止过拟合）
  A_drop = A · m / (1 - p),  m ~ Bernoulli(1-p),  p=0.1
  论文原文: "Higher drop rates were found to cause over-regularization,
            thereby impairing fine-grained road detail extraction."

Step 5: 通道注意力加权
  Y = X · A_drop
```

### 5.3 PyTorch 实现

```python
class EFDCA(nn.Module):
    """
    增强型频域通道注意力（Enhanced Frequency-Domain Channel Attention）

    参考 DSWFNet 论文 Fig.5，公式 14-18
    基于 SE 通道注意力改进，增加 GAP+GMP 双路径 + 3 层 1×1 卷积

    用于频率分支的各阶段，将频域特征通道数从 n_in 映射到 n_out。
    """
    def __init__(self, in_channels, out_channels, reduction=16, drop_path_rate=0.1):
        super().__init__()
        self.drop_path_rate = drop_path_rate

        # Step 1+2: GAP + GMP → 拼接 → 压缩维度
        # in: (B, in_channels, H, W) → out: (B, 2*in_channels, 1, 1)
        hidden_channels = max(in_channels // reduction, 8)

        self.fc1 = nn.Conv2d(in_channels * 2, hidden_channels, kernel_size=1, bias=False)
        self.bn1 = nn.BatchNorm2d(hidden_channels)
        self.act1 = nn.ReLU(inplace=True)

        self.fc2 = nn.Conv2d(hidden_channels, hidden_channels, kernel_size=1, bias=False)
        self.bn2 = nn.BatchNorm2d(hidden_channels)
        self.act2 = nn.ReLU(inplace=True)

        self.fc3 = nn.Conv2d(hidden_channels, out_channels, kernel_size=1, bias=False)
        self.bn3 = nn.BatchNorm2d(out_channels)

        self.sigmoid = nn.Sigmoid()

    def forward(self, x):
        # Step 1: 全局统计
        x_avg = F.adaptive_avg_pool2d(x, output_size=1)   # (B, C, 1, 1)
        x_max = F.adaptive_max_pool2d(x, output_size=1)     # (B, C, 1, 1)

        # Step 2: 拼接
        x_cat = torch.cat([x_avg, x_max], dim=1)             # (B, 2C, 1, 1)

        # Step 3: 三层 1×1 卷积
        w = self.act1(self.bn1(self.fc1(x_cat)))          # (B, C/r, 1, 1)
        w = self.act2(self.bn2(self.fc2(w)))               # (B, C/r, 1, 1)
        w = self.bn3(self.fc3(w))                          # (B, C_out, 1, 1)
        w = self.sigmoid(w)

        # Step 4: DropPath（训练时随机丢弃部分注意力权重）
        if self.training and self.drop_path_rate > 0:
            batch_size = x.size(0)
            keep_prob = 1.0 - self.drop_path_rate
            # 逐样本随机掩码（与 Dropout 类似，但作用在通道注意力权重上）
            mask = torch.rand((batch_size, 1, 1, 1),
                             device=x.device, dtype=x.dtype) < keep_prob
            w = w * mask.float() / keep_prob

        # Step 5: 通道注意力加权
        return x * w
```

## 六、改进点三：BCAM 双向交叉注意力模块

### 6.1 改进目的

DSWFNet 论文原文指出现有跳跃连接的简单相加（`d3 = decoder3(e3) + e2`）存在两个核心问题：

1. **语义异质性（Semantic heterogeneity）**：空间域特征和频域特征处于不同的语义空间，直接相加会导致特征对齐不良。

2. **方向性信息丢失**：简单相加无法有效融合频域特征中的方向性信息。

BCAM（Bidirectional Cross-Attention Module）通过双向交叉注意力显式对齐空间域和频域特征的语义空间，并保留方向性信息。

### 6.2 数学原理

BCAM 的核心思想（来自 DSWFNet 论文原文，公式 19-25，Fig.6）：

```
给定空间域特征 X_s ∈ R^(B×C×H×W) 和频域特征 X_f ∈ R^(B×C×H×W)：

Step 1: 语义空间对齐（共享 1×1 卷积投影）
  F_s = W_p · X_s
  F_f = W_p · X_f
  其中 W_p 为共享的投影权重

Step 2: 双路径交叉注意力
  水平分支（行方向）：
    Q_s, K_f, V_f ∈ R^(B×h×W×(H·d))   # 从频域特征得到 query
    Q_f, K_s, V_s ∈ R^(B×h×H×(W·d))   # 从空域特征得到 query
    A_h = Softmax((Q_s · K_f^T)/τ + P_w)   # 水平注意力权重
    A_v = Softmax((Q_f · K_s^T)/τ + P_h)   # 垂直注意力权重
  τ: 可学习温度参数，P_w/P_h: 可学习位置偏置

Step 3: 上下文聚合
  Z_h = A_h · V_f + Q_s          # 水平分支 + 残差
  Z_v = A_v · V_s + Q_f          # 垂直分支 + 残差

Step 4: 门控融合
  Z = σ(g) · Z_h + (1 - σ(g)) · Z_v
  其中 g 为可学习门控参数（控制水平/垂直方向信息的比例）

Step 5: 最终输出（残差连接 + MLP 细化）
  Y = MLP(W_o · Z) + X_s + X_f
```

### 6.3 PyTorch 实现（简化版，保留核心原理）

```python
class BCAM(nn.Module):
    """
    双向交叉注意力模块（Bidirectional Cross-Attention Module）

    参考 DSWFNet 论文 Fig.6，公式 19-25
    将空间域特征和频域特征通过双向交叉注意力深度融合。

    本实现为简化版本，保留核心的水平/垂直双路径和门控融合机制。
    完整实现请参考 DSWFNet 官方代码。
    """
    def __init__(self, channels, num_heads=4):
        super().__init__()
        self.num_heads = num_heads
        self.channels = channels
        head_dim = channels // num_heads

        # 共享投影（语义空间对齐）
        self.proj = nn.Conv2d(channels, channels, kernel_size=1, bias=False)

        # 水平分支：行方向的 Query-Key-Value 投影
        self.h_q = nn.Conv2d(channels, channels, kernel_size=1)
        self.h_kv = nn.Conv2d(channels, channels * 2, kernel_size=1)

        # 垂直分支：列方向的 Query-Key-Value 投影
        self.v_q = nn.Conv2d(channels, channels, kernel_size=1)
        self.v_kv = nn.Conv2d(channels, channels * 2, kernel_size=1)

        # 可学习门控参数
        self.gate_fc = nn.Sequential(
            nn.AdaptiveAvgPool2d(1),
            nn.Conv2d(channels, channels // 4, kernel_size=1),
            nn.ReLU(inplace=True),
            nn.Conv2d(channels // 4, 2, kernel_size=1),
        )

        # 输出投影 + MLP 细化
        self.output_conv = nn.Sequential(
            nn.Conv2d(channels, channels, kernel_size=1, bias=False),
            nn.BatchNorm2d(channels),
        )
        self.mlp = nn.Sequential(
            nn.Conv2d(channels, channels * 4, kernel_size=1),
            nn.GELU(),
            nn.Conv2d(channels * 4, channels, kernel_size=1),
        )

    def forward(self, spatial_feat, freq_feat):
        """
        Args:
            spatial_feat: 空间域特征 (B, C, H, W)
            freq_feat: 频域特征 (B, C, H, W)
        Returns:
            融合后的特征 (B, C, H, W)
        """
        B, C, H, W = spatial_feat.shape

        # Step 1: 语义空间对齐
        F_s = self.proj(spatial_feat)
        F_f = self.proj(freq_feat)

        # 残差路径（不经注意力处理的原始特征）
        residual = F_s + F_f

        # ======== 水平分支（Row-wise attention）=======
        Q_s = self.h_q(F_s)                         # (B, C, H, W)
        K_h, V_h = self.h_kv(F_f).chunk(2, dim=1)   # (B, C, H, W) each

        # 重排列为 (B, heads, head_dim, H, W) 进行注意力计算
        Q_s = Q_s.view(B, self.num_heads, head_dim, H, W)
        K_h = K_h.view(B, self.num_heads, head_dim, H, W)
        V_h = V_h.view(B, self.num_heads, head_dim, H, W)

        # 简化注意力：沿 H 维度做全局注意力
        # Q_s: (B, heads, head_dim, H, W) → (B, heads, H, head_dim, W)
        Q_s_perm = Q_s.permute(0, 1, 3, 2, 4)      # (B, heads, H, head_dim, W)
        K_h_perm = K_h.permute(0, 1, 3, 4, 2)      # (B, heads, H, W, head_dim)
        V_h_perm = V_h.permute(0, 1, 3, 2, 4)      # (B, heads, H, head_dim, W)

        # 注意力分数 + Softmax → 注意力权重
        attn_h = torch.softmax(
            (Q_s_perm * K_h_perm).sum(dim=3, keepdim=True) / (head_dim ** 0.5),
            dim=4
        )  # (B, heads, H, head_dim, W)

        # 加权聚合
        out_h = (V_h_perm * attn_h).permute(0, 1, 3, 2, 4).contiguous()
        out_h = out_h.view(B, C, H, W) + Q_s          # 残差连接

        # ======== 垂直分支（Column-wise attention）=======
        Q_v = self.v_q(F_s)
        K_v, V_v = self.v_kv(F_f).chunk(2, dim=1)

        Q_v = Q_v.view(B, self.num_heads, head_dim, H, W)
        K_v = K_v.view(B, self.num_heads, head_dim, H, W)
        V_v = V_v.view(B, self.num_heads, head_dim, H, W)

        Q_v_perm = Q_v.permute(0, 1, 4, 3, 2)      # (B, heads, W, head_dim, H)
        K_v_perm = K_v.permute(0, 1, 4, 2, 3)      # (B, heads, W, H, head_dim)
        V_v_perm = V_v.permute(0, 1, 4, 2, 3)      # (B, heads, W, head_dim, H)

        attn_v = torch.softmax(
            (Q_v_perm * K_v_perm).sum(dim=4, keepdim=True) / (head_dim ** 0.5),
            dim=3
        )  # (B, heads, W, head_dim, H)

        out_v = (V_v_perm * attn_v).permute(0, 1, 3, 4, 2).contiguous()
        out_v = out_v.view(B, C, H, W) + Q_v          # 残差连接

        # ======== 门控融合=======
        # 基于全局特征动态决定水平/垂直分支的融合比例
        gate = self.gate_fc(out_h + out_v)              # (B, 2, 1, 1)
        gate_h, gate_v = gate[:, 0:1], gate[:, 1:2]      # 各 (B, 1, 1, 1)
        gate_h = torch.sigmoid(gate_h)
        gate_v = 1 - gate_h                               # 保证 gate_h + gate_v = 1

        fused = gate_h * out_h + gate_v * out_v           # (B, C, H, W)

        # ======== 输出投影 + MLP 细化 + 残差连接 ========
        out = self.output_conv(fused)
        out = self.mlp(out) + residual                    # 残差连接

        return out


class BCAMFusion(nn.Module):
    """
    替换现有跳跃连接的简单相加。

    在 decoder3 + e2 和 decoder2 + e1 处使用 BCAM 进行深度融合，
    参考 DSWFNet 论文 Fig.1 中 BCAM 模块的作用位置。
    """
    def __init__(self, channels):
        super().__init__()
        self.bcam = BCAM(channels)

    def forward(self, decoder_feat, encoder_feat):
        """
        Args:
            decoder_feat: 解码器上采样后的特征
            encoder_feat: 编码器跳跃连接的特征
        Returns:
            融合后的特征（直接替换原有的 decoder_feat + encoder_feat 相加）
        """
        return self.bcam(decoder_feat, encoder_feat)
```

## 七、改进点四：频域感知损失（FAL）

### 7.1 改进目的

频域感知损失（FAL, Frequency-Aware Loss）在训练损失函数层面引入频域监督，直接在边缘/高频层面提供监督信号，弥补仅靠空间域损失难以精确学习细边界的问题。

此改进**零侵入**现有模型结构，只需在 `loss.py` 中新增一个损失类，在总损失中叠加一个权重项。

### 7.2 数学原理

FreqU-FNet 论文原文（公式 9-14）提出的频域感知损失设计：

```
给定预测图 y_pred 和标签 y_gt：

Step 1: 离散小波变换（DWT）分解
  W(y_pred) = (y_pred_LL, y_pred_LH, y_pred_HL, y_pred_HH)
  W(y_gt)   = (y_gt_LL, y_gt_LH, y_gt_HL, y_gt_HH)

Step 2: 高频子带 L1 损失（仅比较高频成分）
  L_freq = (1/D_H) · Σ_{d=1}^{D_H} |y_pred_H^d - y_gt_H^d|_1
  其中 D_H = 3（LH, HL, HH 三个高频子带）

Step 3: 总损失
  L_total = w_dice · L_dice + w_ce · L_ce + w_freq · L_freq
```

**核心洞察**：低频子带（LL）编码了粗粒度的类别信息，已被标准 Dice Loss 充分覆盖。高频子带（LH, HL, HH）编码了边缘和纹理细节，是草线分割最关键的信息，但标准损失函数对此监督不足。

### 7.3 PyTorch 实现

```python
import pywt


class FrequencyAwareLoss(nn.Module):
    """
    频域感知损失（Frequency-Aware Loss, FAL）

    参考 FreqU-FNet 论文（arXiv:2505.17544），公式 9-14
    通过小波变换（DWT）提取预测图和标签的高频子带，
    计算高频子带之间的 L1 损失，直接监督边缘和纹理细节。

    使用 Daubechies-4 (db4) 小波基：
      - 比 Haar 有更好的正则性，对平滑边缘重建质量更高
      - FreqU-FNet 论文推荐使用 db4
    """
    def __init__(self, wavelet='db4', level=1, weight=0.2):
        super().__init__()
        self.wavelet = wavelet
        self.level = level
        self.weight = weight  # FAL 在总损失中的权重

    def forward(self, pred, target):
        """
        Args:
            pred:   模型预测概率图 (B, 1, H, W) 或 (B, H, W)
            target: 标签掩码   (B, 1, H, W) 或 (B, H, W)
        Returns:
            频域感知损失（标量）
        """
        # 统一维度格式
        if pred.dim() == 4:
            pred = pred[:, 0, :, :]   # (B, H, W)
        if target.dim() == 4:
            target = target[:, 0, :, :]

        batch_size = pred.size(0)
        total_loss = 0.0

        for b in range(batch_size):
            p = pred[b]      # (H, W)
            t = target[b]    # (H, W)

            # Step 1: DWT 分解（level=1 分解一次）
            coeffs_p = pywt.dwt2(p.cpu().detach().numpy(), self.wavelet)
            coeffs_t = pywt.dwt2(t.cpu().detach().numpy(), self.wavelet)

            # pywt.dwt2 返回 (LL, (LH, HL, HH))
            _, (p_LH, p_HL, p_HH) = coeffs_p
            _, (t_LH, t_HL, t_HH) = coeffs_t

            # Step 2: 转回 Tensor（移到 GPU）
            p_LH_t = torch.from_numpy(p_LH).to(pred.device)
            p_HL_t = torch.from_numpy(p_HL).to(pred.device)
            p_HH_t = torch.from_numpy(p_HH).to(pred.device)
            t_LH_t = torch.from_numpy(t_LH).to(target.device)
            t_HL_t = torch.from_numpy(t_HL).to(target.device)
            t_HH_t = torch.from_numpy(t_HH).to(target.device)

            # Step 3: 高频子带 L1 损失
            l_lh = torch.abs(p_LH_t - t_LH_t).mean()
            l_hl = torch.abs(p_HL_t - t_HL_t).mean()
            l_hh = torch.abs(p_HH_t - t_HH_t).mean()
            loss_freq = (l_lh + l_hl + l_hh) / 3.0

            total_loss += loss_freq

        avg_loss = total_loss / batch_size
        return self.weight * avg_loss
```

在 `loss.py` 的 `ConfigurableDualTaskLoss` 中新增 FAL 项：

| BCAM（每处） | ~0.2M | Q/K/V 投影 + MLP + 门控 |
| FrequencyAwareLoss | 0 | 仅推理时计算，不占模型参数 |
| **总新增** | ~0.6~0.8M | 相比 D-LinkNet ~24M 增加 ~2.5~3.3% |

### 10.2 推理时间预估

基于 DSWFNet 论文原文数据（Massachusetts 数据集，256×256 输入）：

| 模型 | 无 TTA 推理时间 | 有 TTA 推理时间 | IoU |
|------|---------------|----------------|-----|
| D-LinkNet | **7.94ms** | 210.37ms | 66.71% |
| DSWFNet | 28.38ms | 765.84ms | **69.12%** |
| **预估：本方案** | ~15~25ms | ~300~500ms | **+1.5~2.5%** |

DSWFNet 论文原文：

> "After disabling MS-TTA, the average latency of D-linkNet decreases from 210.37ms to 7.94ms (~26.5× speedup) and its Peak Alloc from 481.9MB to 375.2MB; DSWFNet decreases from 765.84ms to 28.38ms (~27.0× speedup)."

### 10.3 训练稳定性

DSWFNet 论文原文提到使用 DropPath 正则化提升训练稳定性：

> "A DropPath (stochastic depth) strategy is introduced to improve generalization and training stability, helping to mitigate overfitting." — EFDCA 实现中的 drop_path_rate=0.1 即对应此策略。

建议训练策略：
- 学习率从 `2e-4` 开始（与现有方案一致）
- 使用 warmup（DSWFNet 原论文使用 warmup + cosine annealing）
- DropPath rate 从 0.1 开始，可调

## 十一、预期收益与风险

### 11.1 预期收益

基于论文消融实验数据的保守估计：

| 指标 | 现有模型 | 阶段一（+频率分支） | 阶段一+二（+BCAM） | 阶段一+二+三（+FAL） |
|------|---------|-------------------|-------------------|--------------------|
| 草线 IoU | ~0.704 | +0.8~1.2% → **~0.712~0.716** | +1.0~1.5% → **~0.714~0.719** | +1.2~1.8% → **~0.716~0.722** |
| 植被 IoU | ~0.918 | 基本不变 | 基本不变 | 基本不变 |
| 参数量 | 24M | +0.6M → 24.6M | +0.7M → 24.7M | +0.7M → 24.7M |
| 推理时间（无TTA） | 8ms | +5~10ms → 13~18ms | +7~12ms → 15~20ms | 基本不变 |

### 11.2 潜在风险

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
| 频率分支与空间分支融合不当 | 中 | 使用 BCAM 而非简单相加，BCAM 有残差连接保证至少不退化 |
| 训练不稳定 | 低 | DropPath 正则化，warmup 学习率，预训练 ResNet34 权重冻结初期稳定训练 |
| FAL 权重选择不当 | 低 | 从 0.1~0.3 网格搜索，调参空间小 |
| 对植被分割无提升 | 中 | FAL 主要针对草线高频边缘，植被分割已接近饱和（IoU 0.918），频域改进对植被效果有限 |
