# Damage_Analysis 草方格破损检测系统 — 技术文档（v1.4）

> 模块：`Damage_Analysis/`
> 日期：2026-07-06
> 替代版本：v1.3（被覆盖）
> 目标：基于无人机遥感的草方格破损自动检测，覆盖 9 步流水线 + 5 类产物

---

## 1. 概述

`Damage_Analysis` 是一个面向"草方格沙障破损"的离线检测模块，输入为：
- 草线二值掩码 TIF（`uint8`, 多波段取或）
- 植被二值掩码 TIF（同分辨率）
- 4 波段 RGB-NIR 遥感图（用于可视化）
- ROI 矢量面（Shapefile，可选）
- YAML 配置（`config.yaml`，参数集中管理）

输出（写入 `Damage_Analysis/output/<ROI>_<时间戳>/`）：
1. `damaged_cells.csv` — 逐格检测结果（ID, 坐标, 破损类型, 破损率等）
2. `detection_report.csv` — 单行汇总指标（破损率、连片块数等）
3. `warnings.log` — 校验警告（方向熵超阈、边长偏差等）
4. `G_physical.json` — 物理图（节点 + 边 + 主方向熵等）
5. `G_logical.json` — 逻辑图（草方格 + 邻接 + 破损标记）
6. `report.pdf` — A4 报告（封面 / 摘要 / 饼图 / 热力图 / 详细表 / 附录）
7. `composite.tif` — 半透明合成 GeoTIFF（草线紫 + 植被蓝 + 编号标注）
8. `skeleton.png` — 骨架图（PNG）
9. `heatmap.png` — 破损分布热力图（带指北针）
10. `entropy.png` — **方向熵配图**（新增，论文用，详见 §6.3）

## 2. v1.4 主要变更（相对 v1.3）

| 变更项 | v1.3（旧） | v1.4（新） |
|---|---|---|
| **检测范式** | 图论：枚举物理边 → 围成 4 边形 | 网格投影：估计主方向+L_h/L_v → 投影 L_h×L_v 网格 → 每格对比完整模板 |
| **破损判定依据** | 要求 4 条边都有真实物理边才能识别为 cell | 用"每格内实际草线像素长度 / 4×L" 计算破损率，不依赖完整 4 环 |
| **方向熵处理** | H>2.5 → chaos fallback（仍要求 4 边存在） | H>2.5 → 优先用网格投影（不依赖 4 边） |
| **方向熵配图** | 无 | 新增 `entropy.png`（单峰/双峰/H>阈值 三态对比 + 角度直方图） |
| **合成图输出** | 在 PDF 内嵌 PNG | 同时输出独立 `composite.tif`（GeoTIFF，20MB+） |
| **字体兼容** | 需 CJK | Helvetica + ASCII 标签（无 CJK 字体时降级不报错） |

> **核心动机**：v1.3 在测试数据上仅检出 8 格 vs 人工 160 格（含退化补充），召回率 ~5%。原因是草线破损时物理 4 环几乎不存在。v1.4 改用"先重建参考完整网格，再对比破损"的 template-comparison 范式，这是文献中更稳健的策略（参考 DSWFNet 等基于栅格的破损分析思路）。

## 4. 详细算法

### 4.1 Step 1 — MaskLoader

读取 3 个 TIF：
- `读草线`：band 1-4 做 `or` → 二值掩码 `grass`
- `读植被`：band 1-4 做 `or` → 二值掩码 `veg`
- `读遥感`：band 1-3 → RGB float→uint8
- `读 ROI`：shapefile → Polygon

输出元数据：transform（仿射）、CRS、GSD（m/px）。

### 4.2 Step 2 — DirectionEstimator（v1.4 新增）

**目标**：从草线骨架的拓扑特征估计主方向角 θ 和网格间距 L_h, L_v。

**算法**：
1. 骨架化（`skimage.morphology.skeletonize`）
2. 端点提取（8-邻域前景像素 == 1）
3. 端点对连接（沿 Bresenham 直线连接 0.3m 内的端点）
4. 重新提取节点（deg=1/≥3/2直角）
5. 沿每条边的角度（`atan2(dy, dx)`）建立 360 bin 直方图
6. Top-2 峰 → θ 和 θ+90°，两峰集中度 `top2_ratio`
7. 计算香农熵 `H = -Σ p_i log p_i / log 180`（归一化到 [0,1]）
8. **L_h / L_v 估计**：取所有边的长度，做 KMeans(K=2) 或 1D 模态分解，取两个峰作为主副间距

**输出**：`(θ, L_h, L_v, H, top2_ratio, direction_histogram)`

### 4.3 Step 3 — ReferenceGridBuilder（v1.4 新增）

**目标**：构建覆盖 ROI 的旋转网格模板。

**算法**：
1. 把 ROI 用 (θ, L_h, L_v) 旋转到主方向坐标系
2. 在主方向坐标系里按 L_h × L_v 画网格
3. 把网格交点旋转回原坐标系，得到 `n_rows × n_cols` 个顶点 + `cells = (n_rows−1) × (n_cols−1)` 个矩形
4. 裁剪掉 ROI 外的部分

**输出**：`LogicalGrid` 对象，含 `cells: List[GridCell]`（每格有 4 个顶点坐标、中心、面积）

### 4.4 Step 4 — CellProjector（v1.4 新增，核心）

**目标**：对每个网格单元，统计其内部草线像素数 → 破损率。

**算法（每个 GridCell）**：
1. 把 cell 的 4 个顶点投影回像素坐标（`rowcol(transform, x, y)`）
2. 用 `shapely.geometry.Polygon` + `rasterio.features.rasterize` 拿到 cell 的像素掩码 `cell_mask`
3. 累加 `grass_px = (grass_mask & cell_mask).sum() × gsd²` → 草线面积
4. 累加 `veg_px = (veg_mask & cell_mask).sum() × gsd²` → 植被面积
5. 计算：
   - `L_actual = grass_px / GSD_in_meters_per_pixel` （草线长度 ≈ 像素数 × GSD）
   - `L_total = 4 × L` （理论周长）
   - `damage_ratio = max(0, 1 − L_actual / L_total)`
6. 标记：
   - `damage_ratio < 0.30` → `intact`
   - `damage_ratio ≥ 0.30` 且 `veg_ratio < 0.30` → `true_damage`
   - `damage_ratio ≥ 0.30` 且 `veg_ratio ≥ 0.30` → `false_damage`（被植被遮挡而非破损）

**为什么这能解决 v1.3 漏检问题**：v1.3 要求 4 条物理边完整；v1.4 只统计每格内部草线像素总数。即使 4 条边都断了，只要残留 0.5 条边（=L/2），仍能识别为该格存在并算出 50% 破损率。

### 4.5 Step 5 — DamageClassifier（升级）

基于 v1.3 的连片 BFS + 降级补偿保留：
- `cluster_min_size = 3` 个连续破损格 → 连片破损
- 假性破损格的邻居中有 ≥2 个连片 → 升级为连片

### 4.6 Step 6 — Visualizer

三件产物：

#### 4.6.1 composite.tif（合成图）
- RGBA base = RGB 遥感图
- 半透明紫色 (α=0.5) 叠加草线掩码
- 半透明蓝色 (α=0.3) 叠加植被掩码
- 草方格中心标注 ID（白边 + 颜色按破损类型）
- 写入 GeoTIFF，保留 transform/CRS

#### 4.6.2 heatmap.png（热力图）
- matplotlib 散点：每格中心按破损类型着色
  - 绿=完好，红=真性/连片，黄=假性
- 标签：每点写 cell_id
- 右上角指北针（黑圆 + 黑箭头 + "N"）
- Y 轴反转（图像坐标系）

#### 4.6.3 entropy.png（方向熵配图，NEW，论文用）

详见 §6.3。

### 4.7 Step 7 — ReportGenerator

- `damaged_cells.csv`（含 ID/坐标/破损类型/破损率/veg 遮挡率）
- `detection_report.csv`（含 ROI 面积/总数/完好数/真性破损数/假性破损数/连片块数/最大连片块大小/方向熵/校验模式/警告列表）
- `warnings.log`（校验警告逐行）
- `G_physical.json` / `G_logical.json`（图结构落盘，便于复现分析）
- `report.pdf`（A4，封面 / 摘要表 / 饼图+热力图 / 详细记录表 / 配置附录 / 警告）

```yaml
  veg_mask:   "Damage_Analysis/图论测试/测试植被掩码.tif"

roi:
  name: "测试ROI"

detection:
  grass_size:
    length: 1.5      # ← 关键：实测草方格边长（不是默认 2m）
  thresholds:
    direction_entropy_max: 2.5
    size_deviation: 0.30
    missing_ratio: 0.30
    veg_occlusion_ratio: 0.30
  clustering:
    severe_ratio: 0.70
    cluster_min_size: 3
    enable_downgrade_compensation: true
    min_neighbor_clusters: 2
  validation:
    use_multi_criteria: false
    top2_ratio_min: 0.70
    min_dir_separation_deg: 60
  gap_filling:
    max_fill_steps: 50
```

## 9. 已知限制与未来工作

1. **方向角估计精度依赖骨架质量**：如果草线几乎全退化（H>3），方向估计也会失败；这种情况下应允许人工指定主方向（在 config.yaml 暴露 `manual_axis_angle_deg`）。
2. **网格边界处理**：ROI 边界处的格子会被部分裁剪，应在 `cell_projector` 里加 `coverage_ratio` 字段（只统计 ROI 内比例 ≥50% 的格子）。
3. **真性/假性判别**：当前只用 veg 遮挡率作启发式，可改用"草线残端是否在 cell 内部还是被植被覆盖"的拓扑判据。
4. **PDF 中文支持**：当前因运行环境无 CJK 字体，PDF 使用英文标签。生产部署应安装 `fonts-noto-cjk` 并启用中文标签。
