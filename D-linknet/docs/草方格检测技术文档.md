## 1. 项目概述与研究背景

### 1.1 任务定义

本项目针对沙漠草方格无人机遥感图像的自动化检测与破损评估问题。草方格是荒漠化治理中的重要生物工程措施，由压埋的草本植物形成规则网格状结构（典型规格 2m×2m），用于固定沙面、防止风蚀。其完整性直接影响防风固沙效果，需要定期监测和量化评估。

### 1.2 数据规格

| 项目                 | 描述                                        |
| -------------------- | ------------------------------------------- |
| **图像规格**   | 单张 41389×39576 像素，GSD = 0.006388 m/px |
| **草方格尺寸** | 典型 2m×2m（在图像上约 313×313 像素）     |
| **标注数据**   | 约 30000 平方米                             |
| **掩码类别**   | 0 - 沙地背景；100 - 灌草植被；255 - 草线    |
| **文件大小**   | 约 10GB/张                                  |
| **处理需求**   | 无独立显卡的边缘设备（CPU推理）             |

### 1.3 核心输出指标

| 指标                   | 定义                                 |
| ---------------------- | ------------------------------------ |
| **草方格总数量** | 逻辑对偶图中逻辑节点数（闭合环数）   |
| **草线总长度**   | G_actual 中实际边长度总和（物理米）  |
| **单体破损率**   | $1 - L_{actual} / L_{theoretical}$ |
| **连片退化面积** | 缺失边连通域面积 > 阈值              |
| **灌草遮挡比例** | 缺失边被植被掩膜覆盖的比例           |
| **区域破损率**   | 破损格子数 / 总格子数                |

### 1.4 技术挑战映射

| 挑战         | 技术需求          | 解决方案                                |
| ------------ | ----------------- | --------------------------------------- |
| 细线结构分割 | 高频边界感知      | D-Block (d=1,2,4,8) + Haar DWT 频率分支 |
| 类内不一致   | 极端不平衡优化    | BCE + Dice + FocalBCE 组合损失          |
| 边界模糊     | 频域+空域联合感知 | 频率分支 + DCCA 双向交叉注意力融合      |
| 规则网格识别 | 拓扑图分析        | 双重图拓扑框架 + 方向熵                 |
| 破损归因     | 结构化推理        | 物理图 vs 理论图差异分析                |
| 大图处理     | 内存约束          | 分块推理 + 边界融合                     |
| CPU部署      | 高效推理          | ONNX + OpenVINO INT8 量化               |

### 1.5 核心技术路线演进

```
v1.0: Topo-PIDNet + clDice + Gabor纹理增强
        ↓ PIDNet 细线精度不足，转向 D-LinkNet
v1.x: DinkNet34_less_pool_DualHead + 损失函数筛选实验
        ↓ 草线 IoU 达 0.707，但边界模糊、连通性不足
v2.0: DinkNet34_less_pool_DualHead_Freq + 频域感知增强 ← 当前方案
        + Haar DWT 频率分支 + 线性 1×1 投影 + DCCA + DCFE
        + BCE + Dice + FocalBCE 最优损失组合
```

## 2. 模型架构：频域增强双头D-LinkNet

> 版本说明：当前方案采用 **方案 B（DCCA + DCFE）** 命名。
> 模块 DCCA（Dual-direction Cross-domain Cross-Attention）对应原 BCAM；
> 模块 DCFE（Direction-Consistency Feature Enhancer）对应原 MFE。
> 下文若出现“原 BCAM / 原 MFE”，均指同一实现，仅叙事名称不同。

### 2.1 基础模型选择依据

#### 2.1.1 DinkNet34_less_pool vs DinkNet34 对比

本项目选用 **DinkNet34_less_pool**（跳过 encoder4）而非标准 DinkNet34，原因如下：

| 维度         | DinkNet34（标准）    | DinkNet34_less_pool（本项目采用） |
| ------------ | -------------------- | --------------------------------- |
| Encoder 层数 | Layer1-4（4个stage） | Layer1-3（3个stage，跳过Layer4）  |
| 最终特征通道 | 512                  | 256                               |
| D-Block 通道 | 512                  | 256                               |
| 感受野       | 更大                 | 适中（但更适配草方格尺度）        |
| 参数量       | ~24M                 | 更轻量                            |
| 草线分割精度 | 基准                 | **更优**（已验证）          |

DinkNet34_less_pool 跳过了 ResNet34 的第四个 stage（layer4），最终特征通道数为 256，直接进入 D-Block。相比标准 DinkNet34 的 512 通道，更少的通道数在草方格这类细线分割任务上具有更好的通道利用率，减少过拟合风险。

#### 2.1.2 选型依据

**D-LinkNet 的核心优势（CVPR 2018）：**

- **空洞卷积 D-Block**：在保持高分辨率的同时，通过 dilation=1,2,4,8 的级联空洞卷积将感受野扩大至 71×71，有效建模草方格 2m（约 313 像素）的网格尺度
- **特征转发跳跃连接**：LinkNet 的加法 bypass 机制在融合高低分辨率特征时避免了拼接带来的通道数膨胀，同时保留细线细节
- **ImageNet 预训练 ResNet34**：强大的空间域特征提取基础，迁移到遥感图像无需从头训练

### 2.2 双头架构设计

草线和植被在空间分布上具有显著差异：草线是细线结构（宽度 3-20 像素），植被是斑块状区域。将两个任务解耦为独立解码器，使每个分支能针对各自的空间特性进行优化。

```
输入图像 (3, H, W)
        ↓
  ResNet34 Encoder（ImageNet 预训练，完全共享）
  Layer1 → e1_s (64, H/8, W/8)
  Layer2 → e2_s (128, H/16, W/16)
  Layer3 → e3_s (256, H/32, W/32)
        ↓
  频率分支（Haar DWT → 线性 1×1 投影 → f1/f2/f3，并行处理）
        ↓
  分辨率对齐：f1_a = max_pool2d(f1, 2) → H/4；f2_a = max_pool2d(max_pool2d(f2, 2), 2) → H/8；
             f3_a = max_pool2d(max_pool2d(max_pool2d(f3, 2), 2), 2) → H/16
        ↓
  e1 = e1_s + f1_a (64, H/8, W/8)
  e2 = e2_s + f2_a (128, H/16, W/16)
  e3 = e3_s + f3_a (256, H/16, W/16)
        ↓
  D-Block_more_dilate (d=1,2,4,8,16，256通道，完全共享)
        ↓
  ┌──────────────────┬──────────────────┐
  ↓                              ↓
草线解码器                  植被解码器
Decoder3(256→128)         Decoder3(256→128)
DCCA(d3, e2) + e2         DCCA(d3, e2) + e2
Decoder2(128→64)          Decoder2(128→64)
DCCA(d2, e1) + e1         DCCA(d2, e1) + e1
Decoder1(64→64)           Decoder1(64→64)
FinalDeconv4×4           FinalDeconv4×4
FinalConv3×3×32           FinalConv3×3×32
FinalConv3×3×1            FinalConv3×3×1
        ↓                              ↓
  sigmoid → 草线概率          sigmoid → 植被概率
```

**设计理由：**

- 共享 encoder 保证特征提取能力不因双头而退化，复用了强大的 ImageNet 预训练权重
- D-Block 完全共享，两个任务在中心特征上保持一致的理解
- 独立解码器允许草线和植被各自拥有任务特定的特征精炼路径
- DCCA 融合在跳跃连接处（decoder3+e2、decoder2+e1），替代原有简单相加；实际 `less_pool_DualHead_Freq` 不存在 decoder4（layer4 已跳过），因此图中无 decoder4+e3 级 DCCA

### 2.3 频域感知增强分支

频域分支在 Stem 处与空间分支并行处理原始输入图像，通过 Haar 离散小波变换提取频域特征，弥补 CNN 仅通过空间域卷积隐式学习全局周期性模式的不足。

核心原理（详见第三章）：

- Haar DWT 将图像分解为 LL/LH/HL/HH 四个子带，其中 HL/LH 子带天然捕获水平和垂直边缘，与草线的网格结构精确对齐
- 线性 1×1 投影将 4 个子带分别映射到与 ResNet 各阶段通道数匹配的空间，保留 LL/LH/HL/HH 子带的相对能量
- DCCA 在跳跃连接处进行双向交叉注意力融合，让频域的方向性信息显式注入空间域
- DCFE（新增）在 DCCA 输出后做多方向一致性后处理，通过 4 方向 strip 卷积 + 6 对 cosine similarity 显式强化方向一致性和交叉节点，补充 DCCA row-wise 的跨行方向一致性盲区

### 2.4 完整前向传播流程

```python
def forward(self, x):
    # ======== 空间分支（ResNet34 encoder）=======
    x_conv = self.firstconv(x)          # (B, 64, H/4, W/4)
    x_conv = self.firstbn(x_conv)
    x_conv = self.firstrelu(x_conv)
    x_conv = self.firstmaxpool(x_conv)   # (B, 64, H/4, W/4)
    e1_s = self.encoder1(x_conv)        # (B, 64, H/4, W/4)
    e2_s = self.encoder2(e1_s)          # (B, 128, H/8, W/8)
    e3_s = self.encoder3(e2_s)          # (B, 256, H/16, W/16)

    # ======== 频率分支（Haar DWT + 线性 1×1 投影）=======
    f1, f2, f3 = self.freq_branch(x)    # f1/f2/f3 均为 H/2

    # 对齐频率特征分辨率与空间特征分辨率
    f1_a = F.max_pool2d(f1, 2)                        # H/2 → H/4
    f2_a = F.max_pool2d(F.max_pool2d(f2, 2), 2)       # H/2 → H/4 → H/8
    f3_a = F.max_pool2d(F.max_pool2d(F.max_pool2d(f3, 2), 2), 2)  # H/2 → H/4 → H/8 → H/16

    # ======== 特征融合 =======
    e1 = e1_s + f1_a  # (B, 64, H/4, W/4)
    e2 = e2_s + f2_a  # (B, 128, H/8, W/8)
    e3 = e3_s + f3_a  # (B, 256, H/16, W/16)

    # ======== Center: D-Block (d=1,2,4,8,16) =======
    e3 = self.dblock(e3)

    # ======== 草线解码器 + DCCA/DCFE =======
    d3_g = self.decoder3_grass(e3)
    d3_fused_g = self.dcca3(d3_g, f2_a) + e2
    if self.use_dcfe:
        d3_fused_g = self.dcfe3(d3_fused_g)
    d2_g = self.decoder2_grass(d3_fused_g)
    d2_fused_g = self.dcca2(d2_g, f1_a) + e1
    d1_g = self.decoder1_grass(d2_fused_g)
    out_g = self.finaldeconv1_grass(d1_g)
    out_g = self.finalrelu1_grass(out_g)
    out_g = self.finalconv2_grass(out_g)
    out_g = self.finalrelu2_grass(out_g)
    out_g = self.finalconv3_grass(out_g)
    out_g = torch.sigmoid(out_g)

    # ======== 植被解码器 + DCCA/DCFE =======
    d3_v = self.decoder3_veg(e3)
    d3_fused_v = self.dcca3(d3_v, f2_a) + e2
    if self.use_dcfe:
        d3_fused_v = self.dcfe3(d3_fused_v)
    d2_v = self.decoder2_veg(d3_fused_v)
    d2_fused_v = self.dcca2(d2_v, f1_a) + e1
    d1_v = self.decoder1_veg(d2_fused_v)
    out_v = self.finaldeconv1_veg(d1_v)
    out_v = self.finalrelu1_veg(out_v)
    out_v = self.finalconv2_veg(out_v)
    out_v = self.finalrelu2_veg(out_v)
    out_v = self.finalconv3_veg(out_v)
    out_v = torch.sigmoid(out_v)

    return out_g, out_v
```

## 3. 频域感知增强模块

### 3.1 Haar 小波频率分支

#### 3.1.1 设计目的

草方格图像中，草线是灰度值剧烈变化的区域，对应高频成分；沙地背景是灰度平缓的区域，对应低频成分。传统 CNN 的卷积核在空间域局部操作，难以建模这种全局周期性模式。Haar 小波变换能够将图像分解为四个频率子带，每个子带编码不同方向的频率信息，天然匹配草方格的网格结构。

#### 3.1.2 数学原理

二维 Haar 离散小波变换（DWT）对图像 f(x,y) 的分解公式（来自 DSWFNet 论文原文）：

```
LL(x,y) = 1/2 [f(x,y) + f(x,y+1) + f(x+1,y) + f(x+1,y+1)]
            → 低频近似（整体亮度信息）

LH(x,y) = 1/2 [f(x,y) + f(x,y+1) - f(x+1,y) - f(x+1,y+1)]
            → 水平高频（垂直边缘，响应水平草线）

HL(x,y) = 1/2 [f(x,y) - f(x,y+1) + f(x+1,y) - f(x+1,y+1)]
            → 垂直高频（水平边缘，响应垂直草线）

HH(x,y) = 1/2 [f(x,y) - f(x,y+1) - f(x+1,y) + f(x+1,y+1)]
            → 对角高频（交叉点，响应方格顶点）
```

| 子带 | 数学含义             | 对草方的物理意义                   |
| ---- | -------------------- | ---------------------------------- |
| LL   | 2×2 邻域均值        | 沙地背景和植被的亮度基准           |
| LH   | 水平差分（垂直方向） | **垂直边缘** → 对应水平草线 |
| HL   | 垂直差分（水平方向） | **水平边缘** → 对应垂直草线 |
| HH   | 对角差分             | **交叉点** → 对应方格顶点   |

DSWFNet 论文原文明确指出：

> "Notably, road information is particularly prominent in the **HL sub-bands**, which emphasize **horizontal edge structures**."

这一定论直接可迁移到草方格场景——HL/LH 子带对水平/垂直草线具有最强的响应。

#### 3.1.3 为什么选择 Haar 小波

Haar 小波的四个优势（来自 DSWFNet 论文原文）：

1. **计算代价最低**：仅涉及加减运算，无复杂卷积核
2. **边界锐利**：2×2 局部差分天然保留高频边缘
3. **对局部不连续性敏感**：最适合检测草线这类细线结构
4. **无边界效应**：比 Daubechies、Symlets 等高阶小波更适合细线检测

> "As the order increases, wavelets become smoother... and **exacerbates boundary effects**. Accordingly, we employ the Haar wavelet as the DWT basis: its **low computational cost**, **memory efficiency**, and strong **sensitivity to local discontinuities** (road edges) make it particularly suitable for large-scale remote sensing segmentation." — DSWFNet 论文原文

#### 3.1.4 频率分支架构

```
输入图像 (3, H, W)
        │
        ├──→ [空间分支] ResNet34 encoder → e1 (64), e2 (128), e3 (256)
        │
        └──→ [频率分支] Haar DWT → LL/LH/HL/HH (每通道 4 子带 × 3 通道 = 12 通道)
                    │
                    └──→ Conv2d(12 → 64) → f1
                    └──→ Conv2d(64 → 128) → f2
                    └──→ Conv2d(128 → 256) → f3
```

#### 3.1.5 PyTorch 实现

```python
class HaarWaveletTransform2D(nn.Module):
    """
    二维 Haar 离散小波变换（DSWFNet 论文 Fig.2，公式 1-4）。

    输入: (B, C, H, W) 的 RGB 图像或特征图
    输出: 四个子带 LL, LH, HL, HH
          每个子带分辨率减半，通道数为输入的 4 倍
    """
    def forward(self, x):
        assert x.size(2) % 2 == 0 and x.size(3) % 2 == 0, \
            "Haar DWT requires even spatial dimensions"

        x00 = x[:, :, 0::2, 0::2]  # 左上
        x01 = x[:, :, 0::2, 1::2]  # 右上
        x10 = x[:, :, 1::2, 0::2]  # 左下
        x11 = x[:, :, 1::2, 1::2]  # 右下

        LL = (x00 + x01 + x10 + x11) * 0.5
        LH = (x00 + x01 - x10 - x11) * 0.5
        HL = (x00 - x01 + x10 - x11) * 0.5
        HH = (x00 - x01 - x10 + x11) * 0.5

        return LL, LH, HL, HH


class FrequencyBranch(nn.Module):
    """
    频率分支（参考 DSWFNet 论文 Fig.1 右侧 Frequency Branch）。

    对原始输入图像做 Haar DWT 分解，
    将 4 个子带（LL/LH/HL/HH）通过独立卷积映射到与 ResNet34 各阶段
    通道数匹配的表示空间。
    """
    def __init__(self, spatial_channels=(64, 128, 256)):
        super().__init__()
        c1, c2, c3 = spatial_channels

        self.proj_f1 = nn.Conv2d(12, c1, kernel_size=1, bias=True)
        self.proj_f2 = nn.Conv2d(c1, c2, kernel_size=1, bias=True)
        self.proj_f3 = nn.Conv2d(c2, c3, kernel_size=1, bias=True)

        self.haar = HaarWaveletTransform2D()

    def forward(self, x_input):
        LL, LH, HL, HH = self.haar(x_input)
        freq_feat = torch.cat([LL, LH, HL, HH], dim=1)

        f1 = self.proj_f1(freq_feat)
        f2 = self.proj_f2(f1)
        f3 = self.proj_f3(f2)

        return f1, f2, f3
```

### 3.2 DCCA 双向跨域交叉注意力模块

#### 3.2.1 设计目的

DSWFNet 论文原文指出现有跳跃连接的简单相加（`d3 = decoder3(e3) + e2`）存在两个核心问题：

1. **语义异质性（Semantic heterogeneity）**：空间域特征和频域特征处于不同的语义空间，直接相加会导致特征对齐不良
2. **方向性信息丢失**：简单相加无法有效融合频域特征中的方向性信息

DCCA（Dual-direction Cross-domain Cross-Attention，原论文称 BCAM）通过双向交叉注意力显式对齐空间域和频域特征的语义空间，并保留方向性信息。

#### 3.2.2 数学原理

DCCA 的核心思想（来自 DSWFNet 论文原文，公式 19-25，Fig.6；本项目中重命名为 DCCA，机制不变）：

```
Step 1: 语义空间对齐（共享 1×1 卷积投影）
  F_s = W_p · X_s
  F_f = W_p · X_f

Step 2: 双路径交叉注意力
  水平分支（行方向）：
    Q_s: 空间域 Query, K_f: 频域 Key, V_f: 频域 Value
    A_h = Softmax((Q_s · K_f^T)/τ + P_w)
    Z_h = A_h · V_f + Q_s          # 水平分支 + 残差
  垂直分支（列方向）：
    Q_f: 频域 Query, K_s: 空间域 Key, V_s: 空间域 Value
    A_v = Softmax((Q_f · K_s^T)/τ + P_h)
    Z_v = A_v · V_s + Q_f          # 垂直分支 + 残差

Step 3: 门控融合
  Z = σ(g) · Z_h + (1 - σ(g)) · Z_v
  其中 g 为可学习门控参数（控制水平/垂直方向信息的比例）

Step 4: 最终输出（残差连接 + MLP 细化）
  Y = MLP(W_o · Z) + X_s + X_f
```

#### 3.2.3 PyTorch 实现（简化版）

```python
class DCCA(nn.Module):
    """
    双向跨域交叉注意力模块（Dual-direction Cross-domain Cross-Attention）。

    参考 DSWFNet 论文 Fig.6，公式 19-25；本项目重命名为 DCCA，机制不变。
    将空间域特征和频域特征通过双向交叉注意力深度融合。

    本实现为简化版本，保留核心的水平/垂直双路径和门控融合机制。
    """
    def __init__(self, channels, num_heads=4):
        super().__init__()
        self.num_heads = num_heads
        self.channels = channels
        head_dim = channels // num_heads

        self.proj = nn.Conv2d(channels, channels, kernel_size=1, bias=False)

        # 水平分支
        self.h_q = nn.Conv2d(channels, channels, kernel_size=1)
        self.h_kv = nn.Conv2d(channels, channels * 2, kernel_size=1)

        # 垂直分支
        self.v_q = nn.Conv2d(channels, channels, kernel_size=1)
        self.v_kv = nn.Conv2d(channels, channels * 2, kernel_size=1)

        # 可学习门控
        self.gate_fc = nn.Sequential(
            nn.AdaptiveAvgPool2d(1),
            nn.Conv2d(channels, max(channels // 4, 8), kernel_size=1),
            nn.ReLU(inplace=True),
            nn.Conv2d(max(channels // 4, 8), 2, kernel_size=1),
        )

        # 输出投影 + MLP 细化
        self.output_conv = nn.Sequential(
            nn.Conv2d(channels, channels, kernel_size=1, bias=False),
            nn.BatchNorm2d(channels),
        )
        self.mlp = nn.Sequential(
            nn.Conv2d(channels, channels * 4, kernel_size=1),
            nn.GELU(),
            nn.Conv2d(channels * 4, channels, kernel_size=1),
        )

    def forward(self, spatial_feat, freq_feat):
        B, C, H, W = spatial_feat.shape

        F_s = self.proj(spatial_feat)
        F_f = self.proj(freq_feat)
        residual = F_s + F_f

        # 水平分支（行方向注意力，分块处理以控制显存）
        Q_h = self.h_q(F_s)
        KV_h = self.h_kv(F_f)
        K_h = KV_h[:, :C, :, :]
        V_h = KV_h[:, C:, :, :]

        scale = (C ** 0.5)
        chunk_h = 8
        out_h_parts = []

        for h_start in range(0, H, chunk_h):
            h_end = min(h_start + chunk_h, H)
            Q_chunk = Q_h[:, :, h_start:h_end, :]
            K_chunk = K_h[:, :, h_start:h_end, :]
            V_chunk = V_h[:, :, h_start:h_end, :]

            Q_c2 = Q_chunk.reshape(B * (h_end - h_start), C, W)
            K_c2 = K_chunk.reshape(B * (h_end - h_start), C, W)
            V_c2 = V_chunk.reshape(B * (h_end - h_start), C, W)

            attn = torch.softmax(
                torch.bmm(Q_c2.transpose(1, 2), K_c2) / scale, dim=-1)
            out_row = torch.bmm(V_c2, attn.transpose(1, 2))
            out_row = out_row.transpose(1, 2).reshape(B, -1, h_end - h_start, W)
            out_h_parts.append(out_row)

            del Q_c2, K_c2, V_c2, attn, out_row
            torch.cuda.empty_cache()

        out_h = torch.cat(out_h_parts, dim=2)
        del out_h_parts, Q_h, K_h, V_h
        torch.cuda.empty_cache()
        out_h.add_(spatial_feat)

        # 垂直分支（原地门控）
        gate_v = (self.v_q(F_s) + self.v_kv(F_f)[:, :C, :, :]).sigmoid_()
        out_v = spatial_feat * gate_v + freq_feat * (1 - gate_v)

        # 门控融合
        gate = self.gate_fc(out_h + out_v)
        gh = gate[:, 0:1].sigmoid()
        gv = 1 - gh
        fused = gh * out_h + gv * out_v
        del out_h, out_v, gate, gh, gv, gate_v

        out = self.output_conv(fused)
        del fused
        out = self.mlp(out)
        out.add_(residual)
        return out


class DCCAFusion(nn.Module):
    """
    替换解码器跳跃连接处的简单相加。

    在 decoder3 + e2 和 decoder2 + e1 处使用 DCCA（原 BCAM）进行深度融合。
    use_dcca=True  → 调用 DCCA 跨域交叉注意力（方案 B 默认）
    use_dcca=False → 退化为 decoder_feat + encoder_feat（消融实验对照）
    """
    def __init__(self, channels, use_dcca=True):
        super().__init__()
        self.bcam = BCAM(channels)

    def forward(self, decoder_feat, encoder_feat):
        return self.bcam(decoder_feat, encoder_feat)
```

### 3.3 FAL 频域感知损失

#### 3.3.1 设计目的

频域感知损失（FAL, Frequency-Aware Loss）在训练损失函数层面引入频域监督，直接在边缘/高频层面提供监督信号，弥补仅靠空间域损失难以精确学习细边界的问题。

此改进零侵入现有模型结构，只需在 `loss.py` 中新增一个损失类，在总损失中叠加一个权重项。

#### 3.3.2 数学原理

FreqU-FNet 论文原文（公式 9-14）提出的频域感知损失设计：

```
给定预测图 y_pred 和标签 y_gt：

Step 1: 离散小波变换（DWT）分解
  W(y_pred) = (y_pred_LL, y_pred_LH, y_pred_HL, y_pred_HH)
  W(y_gt)   = (y_gt_LL, y_gt_LH, y_gt_HL, y_gt_HH)

Step 2: 高频子带 L1 损失（仅好比高频率成分）
  L_freq = (1/D_H) · Σ_{d=1}^{D_H} |y_pred_H^d - y_gt_H^d|_1
  其中 D_H = 3（LH, HL, HH 三个高频子带）

Step 3: 总损失
  L_total = w_dice · L_dice + w_ce · L_ce + w_freq · L_freq
```

**核心洞察**：低频子带（LL）编码了粗粒度的类别信息，已被标准 Dice Loss 充分覆盖。高频子带（LH, HL, HH）编码了边缘和纹理细节，是草线分割最关键的信息，但标准损失函数对此监督不足。

#### 3.3.3 PyTorch 实现

```python
class FrequencyAwareLoss(nn.Module):
    """
    频域感知损失（Frequency-Aware Loss, FAL）

    参考 FreqU-FNet 论文（arXiv:2505.17544），公式 9-14
    通过小波变换（DWT）提取预测图和标签的高频子带，
    计算高频子带之间的 L1 损失，直接监督边缘和纹理细节。

    使用 Daubechies-4 (db4) 小波基：
      - 比 Haar 有更好的正则性，对平滑边缘重建质量更高
      - FreqU-FNet 论文推荐使用 db4
    """
    def __init__(self, wavelet='db4', level=1, weight=0.2):
        super().__init__()
        self.wavelet = wavelet
        self.level = level
        self.weight = weight

    def forward(self, pred, target):
        if pred.dim() == 4:
            pred = pred[:, 0, :, :]
        if target.dim() == 4:
            target = target[:, 0, :, :]

        batch_size = pred.size(0)
        total_loss = 0.0

        for b in range(batch_size):
            p = pred[b].cpu().detach().numpy()
            t = target[b].cpu().detach().numpy()

            coeffs_p = pywt.dwt2(p, self.wavelet)
            coeffs_t = pywt.dwt2(t, self.wavelet)

            _, (p_LH, p_HL, p_HH) = coeffs_p
            _, (t_LH, t_HL, t_HH) = coeffs_t

            p_LH_t = torch.from_numpy(p_LH).to(pred.device)
            p_HL_t = torch.from_numpy(p_HL).to(pred.device)
            p_HH_t = torch.from_numpy(p_HH).to(pred.device)
            t_LH_t = torch.from_numpy(t_LH).to(target.device)
            t_HL_t = torch.from_numpy(t_HL).to(target.device)
            t_HH_t = torch.from_numpy(t_HH).to(target.device)

            l_lh = torch.abs(p_LH_t - t_LH_t).mean()
            l_hl = torch.abs(p_HL_t - t_HL_t).mean()
            l_hh = torch.abs(p_HH_t - t_HH_t).mean()
            loss_freq = (l_lh + l_hl + l_hh) / 3.0

            total_loss += loss_freq

        avg_loss = total_loss / batch_size
        return self.weight * avg_loss
```

FreqU-FNet 论文建议 FAL 权重 `w_freq` 从 **0.1~0.3** 开始搜索。

## 3.4 DCFE 方向一致性特征增强器

### 3.4.1 设计目的

DSWFNet 的 Haar + DCCA 链路已覆盖方向性的主要来源——Haar 固定数学滤波提供方向性子带分解，DCCA（原 BCAM）通过 row-wise / column-wise attention 做空间-频域跨域信息聚合。但这两者存在一个共同的盲区：**它们都不编码"方向一致性"信号**。

具体来说：

| 现有模块      | 方向性机制                            | **未覆盖的信号**                                                                 |
| ------------- | ------------------------------------- | -------------------------------------------------------------------------------------- |
| Haar DWT      | `HL=垂直差分`，`LH=水平差分`      | 仅编码"哪个子带对应哪个方向"，不编码"多个方向之间的关系"                               |
| DCCA row-wise | 沿 W 维度聚合，同行内跨列 attention   | **第 h 行的 attention 和第 h+1 行的 attention 完全独立**，跨行方向一致性无法建模 |
| DCCA gate     | `gate_v × F_s + (1-gate_v) × F_f` | 门控是频域/空域的加权混合，不编码几何方向一致性                                        |

DCFE（Direction-Consistency Feature Enhancer，原论文称 MFE）来自 FERDNet 论文（arXiv:2504.19467，公式 4-6，Fig.3），通过**4 方向 strip 卷积 + 6 对 cosine similarity**显式编码这一缺失的信号。

### 3.4.2 数学原理

FERDNet 论文原文（公式 4-6）定义 DCFE（本项目重命名自 MFE）：

```
Step 1: 4 方向 strip 卷积（提取各方向特征）
  F_v   = Conv_vert(F_in)      # 1×3 垂直 strip → 响应水平方向边缘（垂直草线）
  F_h   = Conv_horiz(F_in)     # 3×1 水平 strip → 响应垂直方向边缘（水平草线）
  F_ro  = Conv_diag1(F_in)     # 1×3 strip（斜向核）→ 右上-左下对角
  F_lo  = Conv_diag2(F_in)     # 1×3 strip（斜向核）→ 左上-右下对角

Step 2: 6 对 cosine similarity（度量方向间相似性）
  W = σ(  cos(F_v, F_h)   + cos(F_v, F_ro)  + cos(F_v, F_lo)
                + cos(F_h, F_ro) + cos(F_h, F_lo)
                + cos(F_ro, F_lo) )

Step 3: 方向一致性注意力（加权回原特征，残差连接）
  F_out = W · F_in + F_in
```

**为什么 6 对相似度能编码方向一致性和交叉点？**

6 对 cosine similarity 的求和，对不同场景产生不同的响应模式：

| 场景                                       | 各方向 strip 特征                 | W（求和响应）                  | MFE 强化效果                                     |
| ------------------------------------------ | --------------------------------- | ------------------------------ | ------------------------------------------------ |
| **草线中央**（同方向像素）           | 同方向 strip 响应强，跨方向响应弱 | 中等偏高                       | 强化草线连续性，抑制断裂                         |
| **交叉节点**（方格顶点，多方向汇聚） | 四个方向 strip 全部响应强         | **极高**（6 对全部高分） | **显式强化交叉点**，这是 BCAM 完全做不到的 |
| **断裂/空洞**                        | 无任何方向 strip 响应             | 低                             | 抑制误检（断裂处不强化）                         |
| **背景/噪声**                        | 随机方向响应，无一致性            | 低                             | 抑制噪声区域                                     |

这与 BCAM 的 Q-K attention 有本质区别：**BCAM 编码"空间-频域相关性"，MFE 编码"同特征图内 4 方向两两之间的几何相似性"**——两者是正交的信号。

### 3.4.3 为什么 MFE 能做到 BCAM 做不到的事

BCAM 的方向性是"通过 Q-K 跨域注意力隐式产生的副产物"；MFE 的方向性是"通过 cosine similarity 显式编码的几何无监督度量"：

```
BCAM row-wise attention（已有机制）：
  作用空间：spatial_feat × freq_feat（跨域）
  机制：Q_s[b,h,:] · K_f[b,h,:]^T → 第 h 行内跨列加权聚合
  方向性来源：同行内 W 维度信息流动
  跨行方向一致性：❌（第 h 行和第 h+1 行的 attn 完全独立）

MFE cosine similarity（新增机制）：
  作用空间：单特征图内部（自增强）
  机制：cos(F_v, F_h) + ... + cos(F_ro, F_lo) → 全局 6 对方向相似度求和
  方向性来源：跨方向的几何相似性度量
  跨行方向一致性：✅（6 对涵盖所有方向组合，包含跨行跨列的几何关系）
```

关键点：**MFE 不需要可学习权重来编码方向**，cosine similarity 是数学确定的，sigmoid 只做范围压缩，不存在 EFDCA 那样的梯度消失风险（cosine similarity 输出范围有界 [-1,1]，sigmoid 梯度在有界输入上不会消失）。

### 3.4.4 与 BCAM 的协同关系

MFE 不是 BCAM 的替代品，而是互补的后处理模块。MFE 嵌入在 BCAM 之后，作为空频融合特征的方向性后处理：

```
BCAM.forward 输出 (B, C, H, W)
        │
        ├── 已有信号：
        │   ├── Haar 的方向性子带（固定数学滤波）
        │   ├── BCAM 的空间-频域语义对齐（Q-K 跨域注意力）
        │   └── BCAM 的 row-wise 水平方向信息聚合
        │
        └── 新增信号（MFE 后处理）：
            ├── 垂直方向特征提取（F_v strip conv）
            ├── 水平方向特征提取（F_h strip conv）
            ├── 对角方向特征提取（F_ro, F_lo strip conv）
            ├── 方向一致性度量（6 对 cosine similarity）
            └── 交叉节点显式强化（多方向同时高响应）
```

### 3.4.5 嵌入位置与层级选择

**推荐位置：DCCAFusion.dcca3 输出后，作为后处理层。**

```python

d3_fused_g = self.dcca3(d3_g, f2_a) + e2        # DCCA 融合

# DCFE 后处理（仅 dcca3 层级）

    d3_fused_g = self.dcfe3(d3_fused_g)          # DCFE 方向一致性增强

d2_fused_g = self.dcca2(d2_g, f1_a) + e1
```

**仅在 dcca3 加 DCFE 而不在 dcca2 加的原因：**

| 层级                       | 通道数 | 特征图分辨率    | 适用性分析                                                                                                                  |
| -------------------------- | ------ | --------------- | --------------------------------------------------------------------------------------------------------------------------- |
| `dcca3`（decoder3 阶段） | 128    | H/16（深层语义） | ✅**推荐**——语义最强，strip 卷积的 1×3 核覆盖约 3/16 图像高度，对 2m×2m 规则草方格有足够上下文，交叉节点特征最显著 |
| `dcca2`（decoder2 阶段） | 64     | H/8（浅层细节） | ❌ 不建议——像素太密，strip 卷积局部性过强，难以捕获全局方向一致性；浅层特征的方向信息本身就不如深层稳定                   |

### 3.4.6 与原论文的差异性分析

本小节逐项对照原论文公式与本项目实现，厘清哪些是直接搬运、哪些是原创改进，以支撑论文中的贡献声明。涉及的两篇参考文献标注为 `[3]`（DSWFNet）和 `[4]`（FERDNet）。

#### 3.4.6.1 BCAM 与 DSWFNet Eq.19-25 的逐项对照

DSWFNet 原论文 Fig.6 和公式 19-25 定义了 BCAM 的完整结构。以下逐项对比原论文设计与本项目实现的差异：

| 维度                               | 原论文 BCAM（Eq.19-25）                                                                        | 本项目 BCAM 实现                                                                                | 差异性质                                                                                                                        |
| ---------------------------------- | ---------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| **语义对齐**（Eq.19）        | `F_s = W_p · X_s, F_f = W_p · X_f`，**共享** 1×1 卷积                               | 也用 1×1 proj，但 spatial/freq**各自独立**投影                                           | 独立投影 → 表达更强，属于**架构微调**                                                                                    |
| **多 head**（Eq.20-21）      | 是，h heads，通道拆分为`C = h · d`                                                          | 使用`num_heads=4`，但实际**未做真正的 head 拆分**（直接卷出完整 C 通道）                | 属于**显存优化简化**：减少工程复杂度，保留核心注意力形式                                                                  |
| **水平分支计算**（Eq.22-23） | 全`B×h×W×(H·d)` 重排 → 全 W 维矩阵乘法 attention                                        | **row-chunked 分块**（chunk_h=8），每块内沿 W 做 1 维 attention                           | **核心工程创新**：原版在 256×256 feature map 上显存需求 O(W²·H·C) 约数十 GB，分块后降至 O(chunk·W·C)，单卡 24G 可跑 |
| **垂直分支**（Eq.22-23）     | 完整`B×h×H×(W·d)` 全局 attention + 残差 `Q_f + A_v·V_s`                               | **element-wise gating**：`out_v = F_s · σ(v_q·F_s + v_kv·F_f) + F_f · (1-σ(...))` | **任务适配简化**：草原沙地无强列方向相关性，从 attention 退化为 gate 以提升稳定性                                         |
| **门控机制**（Eq.24）        | **单标量** sigmoid gate `g`，全图共享一个权重                                          | **空间自适应 gate**：GateNet 输出 `(B,2,1,1)`，每个 batch sample 独立一个 `g_h`       | **改进**：从全局标量升级为 per-sample 自适应                                                                              |
| **Cosine 归一化**（Eq.22）   | 是，`Q·K^T / τ`（cosine 空间 + learnable temperature + learnable positional bias P_h/P_w） | **否**，标准 dot-product + `1/√C` scaling，**无 positional bias**                | 属于**实现简化**，但 positional bias 的作用在道路提取任务中并不显著                                                       |
| **残差连接**（Eq.25）        | `+ X_s + X_f` 双加                                                                           | 中间`add_(spatial_feat)` + 末端 `add_(residual)` 双加                                       | 等价                                                                                                                            |

**核心原创贡献摘要（BCAM 部分）**：

1. **行分块注意力（row-chunked attention）**：显存从 O(W²·H·C) 降至 O(chunk·W·C)，是纯工程级创新
2. **垂直分支简化为 element-wise gating**：从理论 O(HW·C) attention 退化为 O(HW·C) 的 gate，**针对草方格长程横向依赖为主的特点做任务适配**
3. **per-sample 自适应 gate**：从原论文全局标量升级

#### 3.4.6.2 MFE 与 FERDNet Eq.1-2 的逐项对照

FERDNet 原论文 Fig.3 和公式 1-2 定义了 MFE 的完整结构。以下逐项对比：

| 维度                   | 原论文 MFE（Eq.1-2）                                                | 本项目 MFE 设计                                                                                                    | 差异性质                                                                                   |
| ---------------------- | ------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------ |
| **Strip 卷积核** | 同一个 1×3 卷积核**旋转 4 个方向** → 4 个 strip（共享权重） | **4 个独立 strip 卷积核**（1×3、3×1、两个斜向核），各自独立可学习                                          | 独立核参数 → 表达更强，属于**任务适配**                                             |
| **方向定义**     | v / h / ro / lo（旋转同核，本质是"同一特征图的不同方向投影"）       | **正交四方向（0°/90°）+ 对角两方向（45°/135°）**，精确匹配草方格网格的**正交 + 45° 交叉节点**结构 | **任务驱动创新**：草方格是严格正交网格 + 规则交叉点，比道路的任意方向更规律          |
| **方向数量**     | 4 个（v/h/ro/lo）                                                   | 4 个（与原论文一致，但核独立）                                                                                     | 一致                                                                                       |
| **相似度组合**   | 6 对求和（Eq.1）                                                    | 6 对求和（保留）                                                                                                   | 完全一致                                                                                   |
| **归一化**       | sigmoid                                                             | sigmoid（保留）                                                                                                    | 一致                                                                                       |
| **MFE 位置**     | VAN-tiny backbone 内，**3 个 MFE 串接**（每 stage 一个）      | **只在最深 BCAM 输出后插入 1 次**，作为 BCAM 的后处理                                                        | **架构级创新**：与 BCAM 形成"跨域对齐 → 同域方向一致性"的两阶段流水线，计算效率更高 |
| **触发条件**     | FERDNet 的 MFE 独立存在，承担方向性增强全部职责                     | MFE 是 BCAM 的**补偿模块**，BCAM 覆盖部分方向性，MFE 只补其盲区                                              | 架构分工明确                                                                               |
| **残差**         | `Y = W × F_in + F_in`（Eq.2）                                    | 保留（与原论文一致）                                                                                               | 一致                                                                                       |

**核心原创贡献摘要（MFE 部分）**：

1. **任务驱动的方向定义**：从"道路的任意方向"重新定义为"草方格严格正交 + 45° 对角"，4 个独立 strip 核替代旋转共享核
2. **与 BCAM 的串联架构**：MFE 不再是独立模块，而是 BCAM 的方向一致性补偿器，明确分工（BCAM = 跨域语义对齐，MFE = 同域方向一致性）
3. **插入点选择**：原论文 3 次级联，你的版本只在最深 BCAM 输出后插入 1 次（避免冗余计算）

#### 3.4.6.3 三个维度的原创贡献总结

综合以上对照，从三个维度评估"不仅仅是搬运"的支撑强度：

| 评估维度               | BCAM                                               | MFE                                           |
| ---------------------- | -------------------------------------------------- | --------------------------------------------- |
| 直接搬运的公式比例     | ~50%（共享 proj / softmax attention / MLP / 残差） | ~70%（4 strip + 6 pair cos + sigmoid + 残差） |
| **工程创新**     | 行分块显存优化 + 垂直简化 gate + per-sample gate   | 与 BCAM 串联 + 单次插入 + 独立 strip 核       |
| **架构创新**     | 频域-空间双路融合的任务适配                        | 后置 BCAM 的方向一致性补偿流水线              |
| **任务域创新**   | 道路 → 草方格，特征语义空间完全不同               | 道路 → 草方格，正交网格而非任意方向          |
| **可发表模块名** | 可（需搭配透明引用）                               | 可（需搭配透明引用）                          |

### 3.4.7 模块重新命名方案

本小节提供四套命名方案，并推荐最适合论文发表的选项。

#### 3.4.7.1 方案 A（推荐）：Grid-BCAM + Grid-MFE

**核心思路**：保留原名，加任务级前缀 Grid，清晰表明"针对草方格任务的特化版本"，审稿人可一键追溯到 [3][4]。

| 原名 | 新名                                       | 含义                           |
| ---- | ------------------------------------------ | ------------------------------ |
| BCAM | **Grid-BCAM**（网格双向交叉注意力）  | 面向网格结构的双向交叉注意力   |
| MFE  | **Grid-MFE**（网格多方向一致性增强） | 面向网格结构的多方向一致性增强 |

**论文正文推荐措辞模板**：

> We adopt the Bidirectional Cross-Attention Module (BCAM) from [3] and tailor it to grid-pattern segmentation. Our **Grid-BCAM** introduces three engineering improvements: (1) **row-chunked attention** to reduce memory footprint from O(W²·H·C) to O(chunk·W·C); (2) simplified vertical branch via **element-wise gating** for stable training on elongated horizontal patterns; (3) **per-sample adaptive gate** instead of the global scalar in [3].
>
> To compensate for Grid-BCAM's inherent blind spot in modeling cross-row direction consistency, we design **Grid-MFE**, inspired by the Multi-Angle Feature Enhancement module in [4]. Unlike [4] which uses a single 1×3 kernel rotated into four directions, Grid-MFE uses **four independent strip kernels** (1×3, 3×1, and two diagonal) tailored to the orthogonal grid + 45° intersection structure of straw-checkerboards. Grid-MFE is placed **after the deepest Grid-BCAM output** as a post-processing compensator.

#### 3.4.7.2 方案 B：DCCA + DCFE

**核心思路**：完全自创缩写，体现更多原创感，但需在正文清晰说明"inspired by"。

| 原名 | 新名                   | 全称                                                              |
| ---- | ---------------------- | ----------------------------------------------------------------- |
| BCAM | **DCCA**（DCCA） | Dual-direction Cross-domain Cross-Attention（双向跨域交叉注意力） |
| MFE  | **DCFE**（DCFE） | Direction-Consistency Feature Enhancer（方向一致性特征增强器）    |

**论文正文推荐措辞模板**：

> We design a **Dual-direction Cross-domain Cross-Attention (DCCA)** module, whose core cross-attention mechanism is inspired by BCAM [3] but differs in three engineering innovations: row-chunked attention for memory efficiency, vertical-branch simplification via element-wise gating, and per-sample adaptive gating.
>
> We further design a **Direction-Consistency Feature Enhancer (DCFE)**, motivated by the cosine-similarity-based direction encoding in MFE [4]. DCFE uses four independent strip kernels instead of [4]'s single rotated kernel, and is placed post-DCCA as a direction-consistency compensator.

#### 3.4.7.3 方案 C：组合名 GDFF / CDCF / DCFA

**核心思路**：将 BCAM + MFE 的组合作为单一新模块命名，强调组合创新，适合"双流融合"为主线叙事的论文。

| 组合名                 | 全称                                               | 说明                    |
| ---------------------- | -------------------------------------------------- | ----------------------- |
| **GDFF**（推荐） | Grid-aware Dual-domain Feature Fusion              | 网格感知跨域特征融合    |
| **CDCF**         | Cross-Domain Coherent Fusion                       | 跨域一致性融合          |
| **DCFA**         | Dual-stream Cross-Attention with Feature Alignment | 双流交叉注意力+特征对齐 |

**论文正文推荐措辞模板**：

> We propose **Grid-aware Dual-domain Feature Fusion (GDFF)**, which integrates a cross-domain attention mechanism inspired by BCAM [3] and a direction-consistency enhancement mechanism inspired by MFE [4]. The two components are cascaded: Grid-BCAM first aligns spatial-frequency semantics, then Grid-MFE enforces intra-domain grid direction coherence. GDFF explicitly encodes both cross-domain correlation and within-domain geometric consistency, which are orthogonal signals for grid-pattern segmentation.

#### 3.4.7.4 方案 D：BSFCA + MOCA

**核心思路**：名字直观描述模块功能，审稿人能立即理解机制，风险最低。

| 原名 | 新名            | 全称                                                                  |
| ---- | --------------- | --------------------------------------------------------------------- |
| BCAM | **BSFCA** | Bidirectional Spatial-Frequency Cross-Attention（双向空频交叉注意力） |
| MFE  | **MOCA**  | Multi-Orientation Cosine-Attention（多方向余弦注意力）                |

#### 3.4.7.5 最终推荐与选择建议

**本项目已选定方案 B（DCCA + DCFE）**，理由：

1. **项目命名已收敛**：代码、训练配置与模型说明已统一采用 `DCCA` / `DCFE`，继续沿用可避免工程与文档双轨维护成本。
2. **原创表达更强**：DCCA / DCFE 完全独立于原论文缩写，正文中仍可明确写出“inspired by BCAM [3] / MFE [4]”，不牺牲引用透明度。
3. **与当前叙事一致**：本节前文、模型说明、训练脚本已全部使用 DCCA + DCFE，文档、代码、实验记录形成统一口径。

**其他方案仍可作为备选说明**：

- **方案 A（Grid-BCAM + Grid-MFE）**：若后续投稿希望降低命名风险，可回退到“Grid-”前缀方案，保留原论文名称，仅改任务前缀。
- **方案 C（GDFF）**：适合论文主线为“双流融合”的整体架构创新时作为架构名使用。
- **方案 D（BSFCA + MOCA）**：适合求稳，名字功能描述性强，审稿人理解成本最低。

### 3.4.8 FERDNet 消融实验依据

FERDNet 论文在 CHN6-CUG 数据集上的消融实验（Table 5）：

```
Baseline 3 (VAN-tiny + U-Net):    IoU 65.2%
+ ICM 单独:                        IoU 65.4%   (+0.2)
+ MFE 单独:                        IoU 67.0%   (+1.8)
+ MFE + ICM:                      IoU 67.6%   (+2.4)
```

MFE（本项目重命名为 DCFE）单独带来了绝大多数的提升（+1.8 vs ICM 的 +0.2），说明 **MFE 是 FERDNet 中方向性增强的核心组件**。ICM 的作用是补充 MFE 在深层特征稀释后的细节补偿，与 DCCA（原 BCAM）的功能存在重叠。

**重要说明**：在你的模型中，DCFE（原 MFE）的增量预期**低于** FERDNet 原始实验中的 +1.8，因为你的模型已经有 Haar + DCCA 承担了部分方向性增强。具体增量需要实验验证，保守估计在 +0.3~+1.0 区间。DCCA 部分覆盖了 DCFE 的作用，但 DCCA 的 row-wise attention 无法建模跨行方向一致性和交叉节点显式强化——这两点是 DCFE 独有的、DCCA 完全缺失的方向性信号。

### 3.4.9 为什么不引入 ICM

ICM（Inter-level Compensation Module，FERDNet 论文公式 7-9）与 DCCA 功能高度重叠：

| ICM（FERDNet）                   | 你的模型中的对应机制                     |
| -------------------------------- | ---------------------------------------- |
| 高层语义 F_high + 浅层细节 F_low | DCCA 的空间-频域双向交叉注意力           |
| 用欧氏距离差异回填浅层细节       | DCCA 的 Q_s × K_f / Q_f × K_s 跨域交互 |
| F_e = 加权 F_low + F_high + 残差 | DCCA 的门控融合 + 残差连接               |

FERDNet 消融中 ICM 单独仅 +0.2 的原因是"没有 MFE（本项目重命名为 DCFE）产生的方向性好特征，ICM 没有东西可补偿"。在你的模型中，即使 DCFE 有效，ICM 也只是把 DCCA 融合后的特征再做一遍高低层补偿——DCCAFusion 本身就在解码器跳跃连接处做特征补偿，ICM 完全冗余。

## 4. 损失函数设计与训练策略

### 4.1 损失函数池

本项目实现了一个插件式损失函数池，支持草线和植被分支的独立配置。经过对多种损失组合的系统实验筛选，最终版本保留 2 个核心损失函数。

| 损失名称       | 类型     | 英文名         | 论文来源                    |
| -------------- | -------- | -------------- | --------------------------- |
| Dice Loss      | 交叠类   | Dice Loss      | Milletari et al., 2016 — V-Net |
| Focal BCE Loss | 交叉熵类 | Focal BCE Loss | 等价于 Focal Loss（α=0.5）  |

### 4.2 公式详解

#### 4.2.1 Dice Loss

$$
D = \frac{2\sum_i y_i p_i + \epsilon}{\sum_i y_i + \sum_i p_i + \epsilon}, \quad \mathcal{L}_{Dice} = 1 - D
$$

直接优化预测和真值的重叠率，专门解决前景-背景严重不平衡的问题。实现中 $\epsilon = 10^{-6}$（`smooth`），保证分母非零、训练稳定。

#### 4.2.2 Focal BCE Loss

$$
\mathcal{L}_{FocalBCE} = -\frac{1}{N}\sum_i \left[ y_i (1-p_i)^{\gamma} \log(p_i) + (1-y_i) p_i^{\gamma} \log(1-p_i) \right]
$$

等价于 Focal Loss（设 α=0.5），二值分割场景下正负样本等权，通过 $(1-p_t)^\gamma$ 调制因子降低大量简单样本的贡献，让模型聚焦于难例（hard examples）。默认 $\gamma = 2.0$。实验数据表明，FocalBCE 在草线分割上比 Focal（α=0.25）效果显著更优，提升 IoU 约 +2.2%。

### 4.3 实验验证的最优损失配置

通过对多种损失组合的系统训练与评估，最终确定草线和植被分支均采用 **Dice + FocalBCE** 组合：

#### 草线（主任务）

```python
'grass': {
    'Dice': 1.0,
    'FocalBCE': 1.0,     # 关键：FocalBCE 是草线提升的核心
}
```

**实验支撑**：Dice + FocalBCE 组合在草线上稳定达到 IoU=0.701-0.707，显著优于基线 Dice+BCE（仅 0.685，+2.2%）。

**核心发现**：

| 发现              | 数据支撑                                                     | 理论解释                                                       |
| ----------------- | ------------------------------------------------------------ | -------------------------------------------------------------- |
| FocalBCE >> Focal | FocalBCE 配置草线 IoU 稳定在 0.701-0.707，Focal 配置仅 0.673 | Focal 的 α=0.25 对正样本加权过低；FocalBCE 的 α=0.5 等权更合理 |
| FocalBCE > BCE    | FocalBCE 替换标准 BCE 后草线 IoU +2.2%                       | 难例聚焦机制对极稀疏草线更有效                                  |

在筛选实验中，cDice、Tversky、FocalTversky、GridLoss、FAL 等损失对草线均无正向收益（IoU 持平或下降），故最终版本不予保留。

#### 植被（辅助任务）

```python
'veg': {
    'Dice': 1.0,
    'FocalBCE': 1.0,
}
```

植被对损失组合不敏感（全范围波动仅 ±2%），Dice + FocalBCE 组合已足够稳定。


## 5. 网络子结构详解

### 5.1 BasicBlock

```python
class BasicBlock(nn.Module):
    """
    基本残差块（ResNet34 / LinkNet 基础构建单元）
    expansion = 1（输出通道数不变）

    结构：conv1(3×3) → BN → ReLU → conv2(3×3) → BN → (+残差) → ReLU
    """
    expansion = 1

    def __init__(self, inplanes, planes, stride=1, downsample=None, no_relu=False):
        super().__init__()
        self.conv1 = nn.Conv2d(inplanes, planes, kernel_size=3, stride=stride,
                              padding=1, bias=False)
        self.bn1 = nn.BatchNorm2d(planes, momentum=0.1)
        self.relu = nn.ReLU(inplace=True)
        self.conv2 = nn.Conv2d(planes, planes, kernel_size=3,
                              padding=1, bias=False)
        self.bn2 = nn.BatchNorm2d(planes, momentum=0.1)
        self.downsample = downsample
        self.stride = stride
        self.no_relu = no_relu

    def forward(self, x):
        residual = x
        out = self.conv1(x); out = self.bn1(out); out = self.relu(out)
        out = self.conv2(out); out = self.bn2(out)
        if self.downsample is not None:
            residual = self.downsample(x)
        out += residual
        return out if self.no_relu else self.relu(out)
```

### 5.2 D-Block（空洞卷积模块）

```python
class Dblock(nn.Module):
    """
    空洞卷积模块（DinkNet / D-LinkNet 核心创新）

    结构：d=1,2,4,8 四级联空洞卷积
    感受野：71×71（对 256×256 输入）
    通道数：256（对应 DinkNet34_less_pool 的 e3 通道）

    设计理由：
    - 级联结构（而非并行 Atrous Spatial Pyramid Pooling）避免了大量通道拼接
    - dilation=1,2,4,8 的级联确保了从小到大的多尺度感受野
    - 残差连接确保训练稳定
    """
    def __init__(self, channel):
        super().__init__()
        self.dilate1 = nn.Conv2d(channel, channel, kernel_size=3, dilation=1, padding=1)
        self.dilate2 = nn.Conv2d(channel, channel, kernel_size=3, dilation=2, padding=2)
        self.dilate3 = nn.Conv2d(channel, channel, kernel_size=3, dilation=4, padding=4)
        self.dilate4 = nn.Conv2d(channel, channel, kernel_size=3, dilation=8, padding=8)

    def forward(self, x):
        dilate1_out = nonlinearity(self.dilate1(x))
        dilate2_out = nonlinearity(self.dilate2(dilate1_out))
        dilate3_out = nonlinearity(self.dilate3(dilate2_out))
        dilate4_out = nonlinearity(self.dilate4(dilate3_out))
        out = x + dilate1_out + dilate2_out + dilate3_out + dilate4_out
        return out
```

感受野计算（以 dilation=8 为例）：第 4 层感受野 = 8 × (3-1) + 1 = 17，实际有效感受野约为 dilation × kernel_size = 24，四个空洞卷积级联后等效感受野约为 71×71。

### 5.3 DecoderBlock

```python
class DecoderBlock(nn.Module):
    """
    解码器块（LinkNet 风格）

    结构：
    conv1(1×1, 降维) → BN → ReLU
    deconv2(3×3, stride=2, 上采样) → BN → ReLU
    conv3(1×1, 恢复通道) → BN → ReLU

    设计理由：
    - 先降维再上采样，节省计算量
    - 使用转置卷积（ConvTranspose2d）进行上采样，与 LinkNet 原版一致
    - 轻量化设计（参数量小），适合双头模型
    """
    def __init__(self, in_channels, n_filters):
        super().__init__()
        self.conv1 = nn.Conv2d(in_channels, in_channels // 4, 1)
        self.norm1 = nn.BatchNorm2d(in_channels // 4)
        self.relu1 = nonlinearity

        self.deconv2 = nn.ConvTranspose2d(
            in_channels // 4, in_channels // 4, 3, stride=2, padding=1, output_padding=1)
        self.norm2 = nn.BatchNorm2d(in_channels // 4)
        self.relu2 = nonlinearity

        self.conv3 = nn.Conv2d(in_channels // 4, n_filters, 1)
        self.norm3 = nn.BatchNorm2d(n_filters)
        self.relu3 = nonlinearity

    def forward(self, x):
        x = self.conv1(x); x = self.norm1(x); x = self.relu1(x)
        x = self.deconv2(x); x = self.norm2(x); x = self.relu2(x)
        x = self.conv3(x); x = self.norm3(x); x = self.relu3(x)
        return x
```

## 7. 破损检测与归因判定

### 7.1 破损类型定义与量化

| 破损类型 | 定义                | 量化指标          |
| -------- | ------------------- | ----------------- |
| 完好     | 草线完整率 ≥ 90%   | < 10% 边长缺失    |
| 轻度破损 | 草线完整率 70-90%   | 10-30% 边长缺失   |
| 重度破损 | 草线完整率 30-70%   | 30-70% 边长缺失   |
| 完全破损 | 草线完整率 < 30%    | > 70% 边长缺失    |
| 连片破损 | 3+ 连续重度破损格子 | 连通域面积 > 阈值 |

### 7.2 灌草遮挡识别

植被掩膜与缺失边的空间叠加分析：重叠比 > 30% 判定为遮挡，遮挡区域的真/假破损区分。

```python
def classify_damage_reason(self, missing_edge, veg_mask):
    """
    判断缺失边的破损原因

    Args:
        missing_edge_pixels: 缺失边覆盖的像素坐标列表
        veg_mask: 植被二值掩码 (H, W)

    Returns:
        reason: 'occluded_by_shrub' | 'genuine_damage' | 'unknown'
        overlap_ratio: 植被覆盖比例
    """
    missing_pixels = set(missing_edge_pixels)
    veg_pixels = set(np.argwhere(veg_mask > 0))

    overlap = missing_pixels & veg_pixels
    overlap_ratio = len(overlap) / max(len(missing_pixels), 1)

    if overlap_ratio > 0.3:
        return 'occluded_by_shrub', overlap_ratio
    else:
        return 'genuine_damage', overlap_ratio
```

### 7.3 归因判定算法

```
if 植被覆盖比例 > 阈值:
    → Occluded_by_Shrub（遮挡）
elif 连片面积 > 阈值:
    → Area_Destroyed（连片损毁）
elif 破损率 > 70%:
    → Complete_Destruction（完全破损）
elif 破损率 > 30%:
    → Severe_Damage（重度破损）
elif 破损率 > 10%:
    → Minor_Damage（轻度破损）
else:
    → Intact（完好）
```

## 8. 大幅遥感图像处理

### 8.1 分块推理策略

大幅遥感图像（41389×39576 像素）无法一次性加载到显存，需分块处理。

```
原始大图 (H, W)
        │
        ├── 滑动窗口分块（Overlap 边长 10-20%）
        │      patch_size = 1024 × 1024
        │      stride = 819（80% overlap）
        │
        ├── 逐块推理 ──→ 逐块概率图
        │
        ├── 边界加权融合（避免接缝）
        │      中心区域：权重 = 1.0
        │      边缘区域：权重 = 线性渐变 0→1
        │
        └── 融合输出 (H, W) 全分辨率概率图
```

### 8.2 边界融合权重

使用线性渐变权重覆盖相邻块的重叠区域：

```python
def compute_blend_weights(patch_size, overlap):
    """
    计算边界融合权重（线性渐变）

    中心区域权重 = 1.0
    边缘区域权重 = 线性从 0 渐变到 1

    patch_size: 分块大小
    overlap: 重叠像素数（每侧）
    """
    weight = np.ones(patch_size)
    fade_len = overlap

    # 顶部渐变
    for i in range(fade_len):
        weight[i] = i / fade_len
    # 底部渐变
    for i in range(fade_len):
        weight[-(i+1)] = i / fade_len

    return weight
```

## 11. 消融实验与对比模型设计

### 11.1 对比实验设计原则

为保证实验结果的科学性与可发表性，对比模型的选择遵循以下三条原则：

**原则一：受控变量（Controlled Variables）。** 所有对比模型共享同一编码器（ImageNet 预训练 ResNet34），仅在"中间处理策略"上形成差异，从而将 D-LinkNet 系列的核心贡献（D-Block 空洞卷积策略）与其他混淆因素解耦。

**原则二：最小可发表对比集（Minimum Publishable Benchmark）。** 4 个对比模型构成一个正交对照集合，每个模型代表一种典型的中间策略（D-Block 空洞、多尺度 ASPP、标准下采样、标准 Skip），足以覆盖主流技术空间，同时控制实验成本。

**原则三：结果可复现。** 所有模型均使用相同的训练配置：300 epoch、余弦退火学习率（warmup=5, T=295, η_min=2e-7）、Adam 优化器（lr=2e-4）、Dice + BCE 联合损失。差异仅来自模型架构本身。

### 11.2 编码器统一：所有模型共享 ResNet34

这是本对比设计的核心约束：**编码器变量被固定，消除不同编码器引入的性能差异**。

| 模型                     | 编码器                     | 说明                    |
| ------------------------ | -------------------------- | ----------------------- |
| DinkNet34（D-LinkNet34） | ResNet34（ImageNet预训练） | CVPR 2018 原文配置      |
| LinkNet34                | ResNet34（ImageNet预训练） | LinkNet 原文配置        |
| U-Net                    | ResNet34（ImageNet预训练） | encoder 替换为 ResNet34 |
| AFENet                   | ResNet34（ImageNet预训练） | 注意力聚焦特征增强       |

将 U-Net 和 AFENet 的默认编码器（VGG16 / ResNet50 等）替换为 ResNet34，是本实验设计的关键操作。统一编码器后，4 个模型的参数量差异大幅收窄，性能差异主要反映解码器 / 中间层策略的有效性。

### 11.3 对比模型详解

#### 模型 1：DinkNet34（锚点基准，D-LinkNet 家族直接前身）

```
来源：CVPR Workshops 2018，Zhou L. et al.
论文：*D-LinkNet: LinkNet With Pretrained Encoder and Dilated Convolution for High Resolution Satellite Imagery Road Extraction*

  - 编码器：ResNet34（ImageNet 预训练），Layer1-4
  - 中间层：D-Block（dilation=1,2,4,8 级联空洞卷积），通道 512
  - 解码器：LinkNet 风格特征转发 bypass，加法融合
  - 输出通道：1（草线二值分割）

核心贡献：首次在道路/细线分割中系统使用级联空洞卷积，在保持高分辨率的同时
         将感受野扩大至 71×71，适配草方格 2m（约 313 像素）的网格尺度

本项目中的角色：所有后续改进的直接基准，性能差距反映频域增强 / 双头设计的增益
```

#### 模型 2：LinkNet34（解码器策略对照，消融 D-Block 的贡献）

```
来源：IEEE VCIP 2017，Chaurasia A. et al.
论文：*LinkNet: Exploiting Encoder Representations for Efficient Semantic Segmentation*

架构特点：
  - 编码器：ResNet34（ImageNet 预训练），Layer1-4
  - 中间层：无（直接下采样，无 D-Block）
  - 解码器：LinkNet 风格特征转发 bypass，加法融合
  - 输出通道：1

核心贡献：定义"特征转发 bypass"机制，在解码器中通过加法而非拼接融合高低分辨率特征

本项目中的角色：与 DinkNet34 对照，消融 D-Block（空洞卷积）的贡献

```

```
  2. 与 LinkNet34 对照：消融"拼接 skip（concat）vs 加法 skip（add）"的贡献
  → 差值 = 加法 skip 对细线分割的独立增益
```

```
论文：*AFENet: An Attention-Focused Feature Enhancement Network for the Efficient Semantic Segmentation of Remote Sensing Images*
DOI: https://doi.org/10.3390/rs16234392

架构特点：
  - 编码器：ResNet34（ImageNet 预训练）
  - 中间层：注意力聚焦特征增强（AFE）与多尺度上下文增强模块
  - 解码器：注意力聚焦上采样 + 低层特征拼接
  - 输出通道：1

核心贡献：通过注意力聚焦机制强化关键特征通道，并在多尺度上下文增强模块中整合全局信息，提升遥感图像语义分割的精度与效率

本项目中的角色：
  - 与 DinkNet34 对照：ASPP 与 AFENet 注意力增强对细线分割的独立增益
  - AFENet 差值 = 注意力聚焦增强对草方格分割的独立增益

实现来源：使用 `segmentation_models_pytorch`（smp）库，AFENet with ResNet34 encoder
```

### 11.4 本项目完整方法（单头版本）作为锚点

```
完整方法：DinkNet34_less_pool + Freq（Haar DWT 频率分支 + BCAM 融合）
编码器：ResNet34（与上述 4 个模型完全一致）
         ↕ 关键差异：
         1. less_pool（跳过 Layer4，通道 256 vs 512）
         2. Haar DWT 频率分支（高频细节增强）
         3. BCAM 双向交叉注意力（频-空域特征融合）
         4. FAL 频域感知损失

单头版本：ENABLE_DUAL_HEAD = False
         仅输出草线分割掩码，与对比模型在同等的输出形式下做精度对比
         双头版本用于同时输出草线 + 植被，服务完整流程

实验目标：
  - 本方法 vs DinkNet34（锚点） → 频域增强 + less_pool 的独立增益
  - 本方法 vs 所有 4 个模型 → 整体方法在细线分割任务上的优越性
```
